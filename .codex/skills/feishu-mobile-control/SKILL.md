# 飞书移动控制台

目标是把独立飞书应用用成个人远程控制入口，而不是把它做成无审计的聊天机器人。

## 标准流程

1. 判断事件是否值得推送到飞书。
2. 生成摘要、影响、证据编号和建议动作。
3. 判断可用按钮是否低风险。
4. 为每个按钮生成一次性 action id。
5. 接收飞书事件或卡片回调后校验白名单、过期时间和幂等性。
6. 将动作写入控制面决策、任务或审计记录。
7. 向飞书回发执行结果或工作台链接。

## 标准输出

```text
是否推送：
消息级别：
消息摘要：
证据编号：
允许动作：
禁止动作：
是否需要工作台二次确认：
审计写回：
```

## 规则

1. 飞书不能直接下单、撤单、改实盘配置或推进实盘。
2. 高风险动作只创建审批请求。
3. 飞书应用、用户和会话必须白名单。
4. App Secret 和访问令牌不能进入仓库、日志或产物。
5. 飞书消息不是事实源，控制面才是事实源。
