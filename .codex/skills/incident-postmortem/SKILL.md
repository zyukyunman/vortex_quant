# 事故复盘

目标是把事故变成系统改进，而不是只修一次 bug。

## 流程

1. 建立时间线和影响范围。
2. 区分触发原因、根因和放大因素。
3. 记录已执行修复和未完成风险。
4. 生成预防 task、监控项或 skill 更新建议。

## 输出

```text
事故摘要：
影响范围：
时间线：
根因：
修复：
预防动作：
是否需要用户决策：
```

## 规则

1. 不责备智能体或人，聚焦系统缺口。
2. 没有证据的根因只能标为假设。
3. 交易事故必须进入审计日志。
