# 持续运营状态简报

目标不是“每天喊智能体上班”，而是让系统持续运转，并在需要用户关注时生成高密度状态摘要。

## 输入

1. 数据、研究、策略、交易、智能体的最新运行状态。
2. 实盘、模拟盘、影子跟踪状态和 QMT 健康状态。
3. 最新结构化产物和待审批决策。
4. 已阻断、心跳过期、失败或等待人工输入的运行。
5. 最近一次通知是否已送达、是否需要再次提醒。

## 输出

```text
总体状态：
实盘/模拟盘：
数据：
研究：
智能体：
待用户处理：
建议优先级：
是否需要通知：
```

## 规则

1. 先列阻塞和审批，再列正常完成事项。
2. 每个结论必须能点到运行、产物或决策记录。
3. 不展开长日志，只给证据入口。
4. 没有需要人工处理的事项时，不强行打扰用户。
