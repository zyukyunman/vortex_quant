# 交易审查

目标是确认交易链路是否应该继续、暂停或人工处理。

## 流程

1. 确认账户模式和今日是否交易日。
2. 检查数据快照、策略运行、目标组合。
3. 检查 QMT 桥接健康、认证和账户。
4. 核对订单、成交、持仓和对账。
5. 输出正常、部分完成、人工处理或阻断。

## 输出

```text
交易状态：
数据/策略输入：
QMT 状态：
订单/成交/持仓：
风险：
建议动作：
```

## 规则

1. 状态不一致先对账。
2. 关键数据缺失时默认阻断。
3. 实盘动作必须用户确认。
