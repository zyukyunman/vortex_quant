# 运行故障分诊

目标是把失败从“看日志”转成可执行的恢复计划。

## 流程

1. 确认 run 状态、当前 task、最近心跳和错误。
2. 判断失败类型：数据、代码、外部服务、权限、风控、人工等待。
3. 找到最后一个可信产物。
4. 给出重试、修复、阻断、追问用户或事故复盘的建议。

## 输出

```text
分诊结论：
失败类型：
证据：
影响范围：
可恢复点：
下一步动作：
```

## 规则

1. 不重跑已成功且可复用的步骤。
2. 不在根因不明时继续下游。
3. 涉及交易链路时升级执行运营和风险官。
