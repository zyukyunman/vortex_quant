# QMT 执行就绪审查

用于判断一个策略是否可以从回测进入 QMT/MiniQMT shadow、模拟盘或实盘。核心原则：**先验证开盘成交和委托生命周期，再谈自动下单。**

---

## 一、结论分级

| 结论 | 含义 |
|---|---|
| `只可研究` | 仍停留在回测，不能接模拟盘 |
| `可 shadow` | 可以生成目标订单，但不下单 |
| `可模拟盘` | 可以接国金模拟盘或测试账户验证委托/成交 |
| `可人工确认实盘` | 真实账户可生成订单草案，由人确认 |
| `可小资金自动化` | 小资金、硬风控、异常停机后才允许自动 |

默认从 `只可研究` 开始，逐项通过后才能升级。

---

## 二、开盘价假设审查

回测按开盘价成交时，必须额外回答：

1. 买入日是否开盘涨停。
2. 开盘跳空幅度：`open / prev_close - 1`。
3. 高开 3%、5%、8% 后的 open→close 是否仍为正。
4. 如果开盘涨停，回测是否假设成交。
5. 如果集合竞价量不足，是否模拟部分成交。
6. 实盘成交价相对开盘价的滑点如何记录。

没有分钟或逐笔数据时，先做日频折损测试：

```text
开盘涨停：不成交
高开超过阈值：跳过 / 降权 / 加滑点
成交额不足：部分成交
```

---

## 三、交易物理约束

逐笔订单必须检查：

1. 涨停买不到。
2. 跌停卖不出。
3. 停牌不能交易。
4. ST / 风险警示不能主动买入。
5. 100 股整手。
6. 可用现金不足不能买。
7. 可用股份不足不能卖。
8. 单票金额、单日金额、总仓位不得超过风控阈值。

任何交易状态不确定时 fail-closed，不下单。

---

## 四、委托生命周期

模拟盘或实盘必须记录：

1. 订单草案 `OrderIntent`。
2. 实际报单 `OrderSubmitted`。
3. 委托状态：已报、部成、已成、已撤、废单。
4. 成交回报：成交价、成交量、成交金额、成交时间。
5. 拒单原因：非交易时间、价格非法、资金不足、权限不足、涨跌停、停牌等。
6. 撤单结果。
7. 盘后账户持仓与 Vortex 账本是否一致。

不能只记录“下单成功”，必须记录最终是否成交。

---

## 五、收益归因

实盘/模拟盘报告必须拆：

| 项 | 说明 |
|---|---|
| open→close | 回测主 alpha 口径 |
| close→next open | 隔夜段真实持仓风险 |
| 成本 | 佣金、印花税、过户费 |
| 滑点 | 实际成交价 vs 计划开盘价 |
| 未成交 | 目标仓位未达成造成的差异 |
| 现金拖累 | 现金闲置造成的差异 |

回测收益和实盘收益不一致时，先按这张表拆，不要直接归咎于策略失效。

---

## 六、升级条件

进入国金模拟盘前：

1. Mac 虚拟机 / QMT bridge POC 通过。
2. 策略有最新目标持仓。
3. 开盘价折损测试完成。
4. 涨跌停、停牌、ST 数据检查完成。
5. 订单风控阈值明确。

进入真实账户人工确认前：

1. 模拟盘连续运行多个交易日。
2. 委托/成交/撤单回报稳定。
3. 盘后对账无差异。
4. 滑点和未成交可解释。

进入小资金自动化前：

1. 人工确认阶段稳定。
2. 单日最大交易金额、单票最大金额、最大亏损、最大未成交率都有硬阈值。
3. 任意异常自动停机。
