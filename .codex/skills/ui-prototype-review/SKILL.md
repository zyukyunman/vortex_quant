# UI 原型验证

目标是把产品规格变成能看的原型，让用户基于界面反馈，而不是只基于文字反馈。

## 流程

1. 先确认上游流程，例如因子研究操作流程或交易审查流程。
2. 写页面故事板和核心用户问题。
3. 生成静态 HTML 或轻量 React 原型。
4. 检查桌面和手机视口。
5. 用截图或浏览器走查确认是否可用。
6. 将反馈回写产品规格。

## 输出

```text
页面目标：
原型文件：
桌面检查：
手机检查：
发现问题：
需要修订的规格：
```

## 规则

1. 界面文案尽量使用中文。
2. 不做营销页，不做装饰性大卡片。
3. 每个模块必须能对应运行、产物、决策或智能体会议字段。
