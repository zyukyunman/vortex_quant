# 通知与手机触达

目标是让系统主动叫用户回来处理真正重要的事项，而不是要求用户每天主动巡检。

## 流程

1. 判断事件级别：紧急、高、中、低。
2. 判断是否需要飞书通知。
3. 生成短摘要、影响范围、证据入口和建议动作。
4. 判断是否需要重复提醒或升级。
5. 将通知结果写入控制面审计。

## 输出

```text
是否通知：
通知级别：
标题：
影响：
证据：
建议动作：
是否需要确认：
```

## 规则

1. 没有人工动作价值时不打扰。
2. 实盘、模拟盘、QMT、对账异常优先通知。
3. 飞书通知不能替代工作台事实源。
