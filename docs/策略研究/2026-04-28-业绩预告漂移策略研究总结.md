# 业绩预告漂移策略研究总结

关联：[[业绩预告漂移策略测试报告]]、[[策略组合研究框架]]、[[因子研究与评测全流程说明]]、[[研究协作与产物治理]]、[[策略域设计说明书]]

## 先说人话版结论

这套策略做的事情很简单：**每天看上市公司新发的业绩预告，挑那些“利润可能明显变好”的公司；如果整个市场环境也不差，就买入并持有一段时间。**

它不是“全年每天都满仓”的策略，也不是传统的季度选股。它赚的是“好业绩预告发布以后，市场没有立刻完全反应，后面一段时间还会继续上涨”的钱。英文里这类能力常叫 alpha，本文后面统一说成“超额收益能力”。

当前最强版本是 v3，可以翻译成下面这几句话：

```text
只看业绩预告公告以后已经能知道的信息
  + 公告后的下一个交易日才允许买
  + 最多跟踪 40 个交易日
  + 每天最多选 80 只分数最高的股票
  + 市场环境可以做多时尽量满仓
  + 每次交易按 20 bps，也就是 0.20% 成本扣掉
  + 过滤成交额太小的股票
  + 剔除 ST 和风险警示股票
  + 涨停买不到就不假装买到，跌停卖不掉就不假装卖掉
  + 用 5 日/20 日指数状态判断市场是否适合做多
```

## 先把术语翻译成人话

| 文档里的词 | 人话解释 |
|---|---|
| 超额收益能力 / alpha | 不是靠大盘涨，而是策略自己选股赚出来的那部分钱 |
| 市场可做 / risk-on | 大盘环境不差，允许开仓 |
| 市场不适合做多 / risk-off | 大盘环境不好，即使有好公告也先不买 |
| 只用当时可见数据 / PIT | 回测时不能偷看未来才知道的信息 |
| 先用过去选参数，再用未来验证 / walk-forward | 防止在整段历史里反复调参，把历史“背答案” |
| 最大回撤 | 从历史最高点往下跌的最大幅度 |
| Sharpe | 每承担一份波动，换来多少收益 |
| Calmar | 每承担一份最大回撤，换来多少收益 |
| bps | 万分之一，20 bps 就是 0.20% |

## 这轮研究到底解决了什么问题

最初目标是找到一个 A 股策略，尽量高年化、低回撤、可解释、可复用，并且不能依赖命令行临时脚本。研究过程里不断遇到三个现实问题：

1. **低回撤硬目标很难单策略完成**：A 股会出现单日全市场下跌，即使选股很强，也难把单策略最大回撤长期压到 5% 内。
2. **漂亮回测容易来自错误口径**：同日收盘买龙虎榜、涨停强度、公告前收益、未复权价格、未计成本，都会制造不可实盘的收益。
3. **调参很容易过拟合**：全样本不断搜索参数，总能找到漂亮组合，但不一定能在未来继续工作。

因此本轮将策略定位从“单独一个策略就必须极低回撤”改成“负责赚钱的子策略”。也就是说，这个策略主要证明自己有很强的赚钱能力；至于账户整体波动，可以在组合层用现金、债券、红利、对冲或其他策略一起控制。

这个定位改变以后，评价标准也跟着改变：不再只盯着“最大回撤必须小于 5%”，而是同时看年化收益、收益/波动性价比、收益/回撤性价比、未来样本能不能继续有效、交易成本扛不扛得住，以及 1 亿资金能不能买得进去。

## 它是强因子弱模型吗？

先说结论：**还不能说它是纯粹的“强因子弱模型”，更准确是“中等强度事件因子 + 简单规则模型 + 市场门控”。**

这里的“强因子弱模型”可以理解为：因子本身预测力很强，用简单排序、等权、Top-N 也能赚钱；“弱因子强模型”则是单个因子预测力弱，需要复杂模型或大量特征才能挖出收益。

这套策略的模型非常简单，没有机器学习：

```text
业绩预告增长区间 + 公告类型
  → 截面排名
  → Top-N
  → 市场门控
  → 等权/单票上限
```

所以它肯定不是“强模型”。但 forecast surprise 单因子评测显示，它本身只有中等偏正的预测力：

| 预测周期 | IC 均值 | ICIR | 正 IC 占比 |
|---:|---:|---:|---:|
| 1 日 | 0.0290 | 0.179 | 55.93% |
| 5 日 | 0.0234 | 0.147 | 57.29% |
| 20 日 | 0.0135 | 0.075 | 53.40% |
| 40 日 | 0.0277 | 0.164 | 55.82% |
| 60 日 | 0.0530 | 0.296 | 64.79% |

20 日多空均值约 0.284%。这说明业绩预告 surprise 方向是对的，尤其 60 日更明显；但它不是那种 IC 极高、单独一排序就足够解释全部收益的强因子。完整策略的高收益，还来自“公告后持有窗口 + 市场可做才交易 + 事件池分散 + 成本和流动性约束”的组合。

## 策略灵感从哪里来

前期尝试过红利、价值、低波、质量、行业轮动、季度池加技术择时、龙虎榜、涨停强度、热榜共振等路径。结论是：

| 路径 | 问题 |
|---|---|
| 红利/价值/低波/质量 | 解释性强，但单独 alpha 不够强 |
| 季度池 + 周频技术择时 | 能改善节奏，但压不住系统性回撤 |
| 行业轮动 | 当前口径没有形成稳定超额 |
| 龙虎榜同日收盘买 | 可能有未来函数风险 |
| 涨停强度同日收盘买 | 需要分钟或实时事件流，当前日频口径不能安全使用 |
| 热榜共振 | 未显著改善业绩预告主策略 |

这些失败路径提示：继续围绕常见低频因子微调，边际收益很低；需要找**信息增量更强、披露时点明确、能用日频数据验证**的新信号。

业绩预告 `forecast` 正好满足这三个条件：

1. **信息强**：预增、扭亏、高增长直接改变市场对公司利润的预期。
2. **时点明确**：有 `ann_date`，可以做 PIT（Point-in-Time，当时可见）审查。
3. **A 股适配**：公告后信息扩散常常不是一天完成，存在公告后漂移空间。

核心假设可以说得更白一点：

```text
好业绩预告出来以后
  市场不一定第一天就完全反应完
  后面一两个月可能还会继续涨
  但如果整个市场很差，先不硬买
```

1. 回测收益用复权 `open→close`，避免除权除息造成虚假收益。

## 策略规则：每天到底怎么运行

### 第一步：先看有没有新的业绩预告

使用 Tushare `forecast` 数据。核心字段：

| 字段 | 用途 |
|---|---|
| `ts_code` / `symbol` | 股票代码 |
| `ann_date` | 公告日期 |
| `p_change_min` | 预告净利润同比变动下限 |
| `p_change_max` | 预告净利润同比变动上限 |
| `type` | 预告类型，例如预增、扭亏、预减 |

只使用公告已经披露的信息，不使用以后才知道的财报修正。这样做是为了避免“回测偷看未来”。

### 第二步：公告当天不急着买，等下一个交易日

公告可能发生在交易日收盘后、周末或节假日。策略不在 `ann_date` 当天直接假设可交易，而是：

```text
event_trade_date = ann_date 之后第 delay_days 个可交易日
delay_days = 1
```

如果公告日不是交易日，也会先找到公告后的第一个交易日，再按延迟规则进入信号。这样做的原因是防止把公告前或公告当天不可获得的信息放进回测。

### 第三步：给每条公告打分

增长幅度使用 `p_change_min` 与 `p_change_max` 的均值：

```text
avg_growth = mean(p_change_min, p_change_max)
avg_growth = clip(avg_growth, -200, 500)
```

裁剪到 `[-200%, 500%]` 是为了避免极端文本、缺失或异常公告把排序完全污染。

公告类型权重：

| 类型 | 权重 |
|---|---:|
| 预增 | +1.0 |
| 扭亏 | +0.8 |
| 略增 | +0.5 |
| 续盈 | +0.3 |
| 减亏 | +0.2 |
| 略减 | -0.5 |
| 预减 | -0.8 |
| 续亏 | -0.7 |
| 首亏 | -1.0 |

最终事件分数：

```text
event_score = clip(avg_growth, -200, 500) / 100 + type_score
```

例子：如果一家公司预告净利润同比增长区间为 80% 到 120%，公告类型为“预增”，则：

```text
avg_growth = 100
growth_score = 1.0
type_score = 1.0
event_score = 2.0
```

### 第四步：每天把所有候选股票排个队

同一天可能有多家公司公告。策略将当天所有事件分数做截面排名：

```text
signal = event_score.rank(axis=1, pct=True)
```

同一股票同一天如果有多个事件，取最高事件分数。排名使用百分位，方便不同日期之间的事件数量差异。

### 第五步：好公告不是只看一天，而是跟踪 40 个交易日

业绩预告漂移不是一天完成，所以信号不是只持有一天：

```text
hold_days = 40
signal = signal.ffill(limit=hold_days - 1)
```

含义是：公告后第一个可交易日产生信号，之后最多延续 40 个交易日；如果期间出现更新的公告或更高分数，会以新的事件排序覆盖。

### 第六步：太难买卖的股票先过滤掉

默认过滤：

```text
liquidity_window = 20
min_avg_amount = 30000
```

即 20 日平均成交额至少为 30000。这里的 `amount` 延续 Tushare 日线口径，容量测算时按千元换算为元。这个阈值约等于 3000 万元日均成交额，是研究默认值，不代表生产容量上限。

如果以后更看重买卖顺畅，可以观察更高成交额版本：

```text
min_avg_amount = 500000
```

它显著降低 1 亿元资金下最难成交那部分交易的参与率，但因为“先用过去选参数，再用未来验证”时没有稳定选中它，所以暂不替换默认版本。

### 第七步：剔除 ST 和风险警示股票

默认：

```text
exclude_st = true
```

`stock_st` 中出现的股票会从候选池剔除。这样做是为了避免公告后漂移被 ST 特殊交易规则、退市风险或极端流动性扭曲。

### 第八步：判断今天市场适不适合做多

策略不是全年无脑满仓，而是先判断市场状态。当前 v3 默认：

```text
benchmark = 000300.SH
confirmation_indices = 000300.SH, 000905.SH, 000852.SH
momentum_window = 5
support_window = 20
min_momentum = 0
support_break_pct = 0.03
min_risk_on_confirmations = 2
```

对每个确认指数计算：

```text
momentum = close / close.shift(5) - 1
support = rolling_min(close, 20).shift(1)
momentum_ok = momentum >= 0
above_support = close >= support * (1 - 0.03)
```

当至少 2 个核心指数短期走势不差、也没有明显跌破近期支撑，同时沪深300没有跌破自己的支撑条件时，就认为“市场适合做多”。

```text
市场适合做多 = 至少 2 个核心指数确认 + 沪深300没有跌破支撑
```

如果市场不适合做多，即使当天有业绩预告信号，也先不买。这就是策略经常空仓的根本原因：它不是每天都要买，而是“好公告 + 市场环境允许”才买。

### 第九步：市场可以做时，买分数最高的一批

每天在有效信号池中取排名最高的 80 只：

```text
top_n = 80
target_exposure = 1.0
max_weight = min(0.2, 1 / top_n) = 1.25%
```

权重规则：

1. 如果当天市场不适合做多：所有目标权重为 0。
2. 如果当天市场适合做多且有事件池：取 `top_n=80`。
3. 入选股票等权，并归一到 `target_exposure=1.0`。
4. `max_weight` 在当前实现中主要用于限制入选数充足时的初始权重；如果某天入选数量少于 80，归一化会让单票权重高于 1.25%，这是“尽量满仓评估 alpha”的设计取舍。
5. 因此生产化时不能只看 `top_n=80`，还要在持仓 CSV 里检查每日实际持仓数量和单票权重尾部。

### 第十步：买不到就不假装买到，卖不掉就不假装卖掉

早期严格近似口径使用：

```text
limit_list_d.limit == "U" => 不允许新增或加仓
limit_list_d.limit == "D" => 不允许减仓或卖出
```

当前精确复核已经使用更严格口径：

```text
stk_limit + 未复权 open
  open >= up_limit   => 开盘涨停，不允许买入/加仓
  open <= down_limit => 开盘跌停，不允许卖出/减仓

suspend_d.suspend_type == "S"
  => 停牌日不允许买入，也不允许卖出
```

交易阻断不是简单删除股票，而是模拟真实约束：

1. 如果目标权重高于当前权重，但当天买入被阻断，则维持当前权重。
2. 如果目标权重低于当前权重，但当天卖出被阻断，则锁定当前权重。
3. 如果被锁定的卖出仓位导致总仓位超过目标仓位，则其他可调整股票按比例缩放。

这个逻辑比“直接把不可交易股票剔除”更接近真实交易，因为跌停或停牌时，持仓通常不是消失，而是被迫保留。

### 第十一步：按开盘买、收盘算收益，并扣交易成本

收益口径：

```text
return_i,t = adjusted_close_i,t / adjusted_open_i,t - 1
portfolio_return_t = sum(weight_i,t * return_i,t) - turnover_t * cost_bps / 10000
transaction_cost_bps = 20
```

也就是说，信号在当天开盘形成目标持仓，收益只吃当天开盘到收盘，不吃前一晚到开盘的跳空。这是偏保守的日频执行口径，避免把实际很难拿到的隔夜跳空收益算进去。

## 为什么不是每天满仓

`target_exposure=1.0` 容易被误解成“每天都应该满仓”。实际含义是：

```text
只要市场适合做多且有合格事件，目标仓位就是 100%
如果市场不适合做多，则目标仓位是 0%
如果事件池不足、不可交易约束锁仓或涨跌停阻断，实际仓位可能低于或偏离 100%
```

v3 精确口径平均仓位为 52.23%，不是因为只买 50 只中的 30 只，也不是因为人为把仓位限制到 60%；主要原因是市场环境过滤让策略在约一半交易日空仓。

```text

```

| 100000 | 60.45% | -12.56% | 2.41% | 4.72% | 435 | 容量改善 |

| Split | 训练区间 | 训练选中阈值 | 测试区间 | 测试结论 |

### 4. 成本压力

### 6. 弱年份复盘

对 2023 年单独拆月、拆行业、拆贡献，而不是为了修复 2023 去硬调参数。

### 7. 无效路径归档

把失败实验写进文档和 skill，后续除非数据或假设改变，不重复穷举同类路径。

## 哪些东西不能机械复用

| 不应机械复用 | 原因 |
|---|---|
| `forecast` surprise 权重 | 只适合业绩预告事件，不一定适合其他公告 |
| `hold_days=40` | 反映业绩预告漂移窗口，不能直接用于龙虎榜、涨停、新闻 |
| `5/20` 市场门控 | 当前只在业绩预告策略中通过验证，其他策略需要单独 walk-forward |
| `min_avg_amount=30000` | 是研究覆盖优先的默认值，不是所有资金规模的生产容量 |
| 20 bps 成本 | 是研究假设，真实交易需按券商、冲击成本和资金规模重估 |

## 策略当前做对了什么

1. 使用公告后第 1 个交易日，避免公告前收益泄漏。
2. 使用复权 open/close 计算收益，避免除权除息污染。
3. 用 `open→close` 收益，不偷吃隔夜跳空。
4. 加入 20 bps 成本，而不是裸收益。
5. 把市场 risk-off 作为风控条件，避免在明显下行阶段强行持仓。
6. 用 `top_n=80` 分散事件噪声，避免单票主导。
7. 阻断涨停买入和跌停卖出，降低不可成交收益。
8. 保存每日持仓，能复盘到底买了什么、错在哪里。
9. 用 walk-forward 和成本压力降低过拟合风险。
10. 把失败路径写入文档和 skill，减少重复无效实验。

## 当前仍然不足

1. 全历史 `stk_limit` / `suspend_d` 已重跑通过，但仍需用 shadow trading 检查当期公告到达时间、停牌更新延迟和真实开盘成交。
2. 默认 `min_avg_amount=30000` 在 1 亿元本金下 P99 参与率约 8.43%，尾部容量偏高。
3. 北交所、科创板贡献较高，需要确认真实权限、滑点、冲击成本和交易限制。
4. surprise 分数还比较朴素，尚未纳入预告可信度、历史兑现率、行业预期差。
5. 2026 年结果只有年初至 4 月，不能当完整年份。
6. 目前报告能打开，但交互式界面已暂缓，持仓查看先依赖 HTML/JSON/CSV artifact。

## 现在这个时间点，几月启动更合适

我新增了一份“每年从某个月开始运行”的测试。它回答的不是“哪个月份永远开、哪个月份永远关”，而是帮助判断上线节奏：如果每年从这个月第一个交易日才启动，后面到年底、未来 1 个月、未来 3 个月的历史胜率和收益大概怎么样。

3. 5 月可以做影子跟踪或小规模观察，但不建议直接满额上线，因为事件密度和市场环境仍要当天确认。

某日 shadow trading / paper trading（影子跟踪）计划：

uv run vortex strategy earnings-forecast shadow-plan \

`shadow-plan` 只生成目标持仓和交易变化，不代表已经下单或成交。2026-04-24 的样例输出为：目标仓位 100%，持仓 44 只，调仓 45 笔；对应 HTML 报告为 [2026-04-28-业绩预告漂移策略shadow跟踪-20260424.html](2026-04-28-业绩预告漂移策略shadow跟踪-20260424.html)。

开盘卖一容量复核（保守 ask1 下界）：

```bash
uv run vortex strategy earnings-forecast opening-liquidity-review \
  --root /Users/zyukyunman/Documents/vortex_workspace \
  --start 20170101 \
  --end 20260424 \
  --opening-snapshots /path/to/opening_snapshots.csv \
  --output-dir /Users/zyukyunman/Documents/vortex_workspace/strategy \
  --label 2026-05-01-业绩预告漂移策略开盘卖一容量复核 \
  --top-n-values 30,40,50 \
  --position-modes full_equal_selected,capped_with_cash \
  --portfolio-notional 1000000 \
  --capped-max-weight 0.05 \
  --volume-unit shares \
  --format json
```

其中外部开盘快照至少要提供：

```text
date,symbol,open_price,ask1_price,ask1_volume
```

这条复核不是完整撮合器，而是判断“如果只看开盘瞬间卖一挂盘量，100 万级别小资金计划买单到底能覆盖多少”。如果连卖一量都明显不够，就不应把回测里的开盘买入视为稳健假设。

## 上线建议

当前不建议直接实盘满额上线，建议分三步：

1. **立即开始 shadow trading**：每天生成真实目标持仓和交易变化，不下单，检查数据延迟、公告更新、涨跌停、ST、停牌和空仓原因。
2. **5 月可以开始影子跟踪或小资金观察，但不建议直接满额上线**：历史上 5 月启动不差，但当前正处于年报/一季报披露之后，需要先看当期真实事件密度。
3. **更合适的小规模上线窗口是 2026 年半年度业绩预告前后**：6 月底到 7 月更贴近半年度预告窗口，历史启动测试也更支持这个区间。
4. **默认版和安全版并行观察**：默认版保持收益弹性；单票 3% 上限留现金版控制集中度和回撤，更适合先小规模实盘。

上线前硬门槛：

| 门槛 | 要求 |
|---|---|
| 精确可交易性 | 已完成全历史 `stk_limit` / `suspend_d` 重跑；影子跟踪继续验证当期数据延迟和真实成交 |
| 开盘成交可靠性 | 对目标资金规模先跑 `opening-liquidity-review`；至少确认一手可成交率和目标覆盖率不失真 |
| 成本压力 | 30 bps 仍保持 A/S，50 bps 风险可解释 |
| 容量 | 目标资金规模下 P99 参与率最好低于 5% |
| 数据延迟 | shadow trading 中无公告延迟或字段缺失异常 |
| 报告归档 | 每日生成 JSON、HTML、持仓 CSV 和审查结论 |
