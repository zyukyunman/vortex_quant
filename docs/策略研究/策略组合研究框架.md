# 策略组合研究框架

关联：[[2026-04-28-业绩预告漂移策略研究总结]]、[[因子方法论与多周期组合]]、[[因子评价框架]]

阶段性策略结论和 artifact 索引集中维护在：[[策略研究结论档案]]。本文件描述框架和评价协议，避免承载每轮实验的完整结论。

## 一句话结论

Vortex 当前不应该寻找“一个因子直接变成最终策略”，而应该构造可解释的策略组合：业绩预告漂移负责主 alpha，质量/红利/低波负责防守和左尾，短反/筹码/101 残差负责持仓质量，市场状态和执行数据负责仓位与成交。

## 当前主策略锚点

当前唯一实盘候选入口仍是业绩预告漂移：

| 维度 | 当前口径 |
|---|---|
| 资金 | 100 万级优先；1000 万以上走 Top80 口径另评估 |
| 选股 | 业绩预告漂移候选池，100 万级 Top30 |
| 权重 | `capped_with_cash`，单票 5% 上限 |
| 成本 | 20 bps |
| 风控 | 原市场门控、ST/停牌/涨跌停过滤 |
| 执行 | 开盘竞价 AON 口径：买不满就视为未买到 |
| 状态 | live 默认不接入新增二级因子；新增因子先做 shadow |

这条线解决“账户中主要赚钱能力来自哪里”。其他因子先证明能改善它，再讨论是否进入默认链路。

## 组合组件

| 组件 | 候选因子/策略 | 目标 | 接入方式 |
|---|---|---|---|
| 主 alpha | 业绩预告漂移 | 抓公告后漂移收益 | 默认持仓来源 |
| 防守腿 | 质量、红利、自由现金流、低波 | 弱市降低左尾 | 弱市替代持仓或半仓现金替代 |
| 交易腿 | 短反、成交拥挤反转、筹码成本宽度、101 残差 | 改善买入优先级和坏持仓剔除 | 候选池 overlay / 持仓缩放 |
| 状态腿 | 大盘动量、宽度、全市场 PRV、波动率 | 决定 risk-on/risk-off 或半仓 | 仓位缩放、暂停新开仓、切防守腿 |
| 执行腿 | 集合竞价、盘口、分钟容量 | 判断买不买得到、买多少 | AON、补买、排队、未成交记录 |

## 研究优先级

1. **不改变 live 默认规则**：所有新增因子先进入 shadow，除非显著改善收益/回撤/成交。
2. **先组合后上线**：单因子 IC 强只能进入候选，必须通过组合层回测。
3. **先小资金口径再大资金口径**：100 万、1000 万、5000 万、1 亿分别评估，不能用 100 万表现推断 1 亿。
4. **先目标权重再执行折损**：先看目标组合是否有效，再叠加开盘竞价/盘口/容量折损。
5. **先解释暴露再合成**：低波、小市值、低换手、短反转等暴露要标清，不把重复风险源当多因子。

## 策略实验矩阵

| 实验 | 说明 | 验收重点 |
|---|---|---|
| Baseline | 业绩预告漂移当前默认 | 作为所有实验对照 |
| Candidate overlay | 在候选池内用二级因子过滤或加权 | 是否提升 Calmar 且不显著增加换手 |
| Holding overlay | 只对已持仓缩放，不引入新股票 | 是否减少坏持仓和左尾 |
| Defensive switch | 弱市切质量/红利/低波腿 | 是否优于硬空仓和简单半仓 |
| Regime switch | 市场状态改变持有期/权重/开仓节奏 | 是否减少回撤恢复期 |
| Execution haircut | 开盘竞价和分钟容量折损 | 是否仍有可实现收益 |
| Capital sweep | 100 万/1000 万/5000 万/1 亿 | 容量上限和策略降级方案 |

## 策略评价协议

Vortex 后续不再只用“年化 + 最大回撤”判断策略，也不把 Sharpe 当唯一标准。所有新因子、overlay、preset mutation 都必须经过一套可复核的 **Strategy Evaluation Protocol**：

```text
数据与方法有效性 gate
  → 训练/验证/测试与 CPCV
  → 指标阈值评分
  → 稳健性/容量/成本复核
  → promote_challenger / keep_shadow / reject
```

### 指标主次

| 层级 | 指标 | 解释 |
|---|---|---|
| 第一核心 | Calmar / MAR | 年化收益相对最大回撤，回答“每承担 1 单位最大回撤换来多少收益” |
| 第二核心 | Sortino | 只惩罚下行波动，避免把上涨波动当坏事 |
| 辅助参考 | Sharpe | 收益平滑度参考，不单独决定晋级 |
| 左尾风险 | CVaR 5%、worst5d、worst20d | 衡量坏日子、最差一周、最差一个月 |
| 路径质量 | 最大回撤持续天数、恢复天数 | 衡量资金和心理占用 |
| 执行质量 | 容量折损后 Calmar、成交覆盖率、成本敏感性 | 判断理论收益是否可实现 |

`CVaR 5%` 建议在报告里写成 **平均极端亏损**：把每日收益从差到好排序，取最差 5% 的交易日，计算这些坏日子的平均收益。

`worst5d` / `worst20d` 建议写成 **最差 5 日累计收益** / **最差 20 日累计收益**：在所有滚动 5 日或 20 日窗口中，找到累计收益最低的一段。它分别回答“最坏一周会亏多少”和“最坏一个月会亏多少”。

### Hard gate

任何策略命中以下任一项，直接 `reject`：

| gate | 具体标准 |
|---|---|
| PIT | 使用未来公告、未来财务、未来复权、未来成分或未来成交信息 |
| 数据覆盖 | 核心字段覆盖低于 70%，且没有明确降级解释 |
| 成本 | 未计当前 preset 对应交易成本 |
| 交易约束 | 未处理涨跌停、停牌、ST/风险警示、整手约束 |
| 参数透明 | 只展示最优参数，不展示邻域、分段或样本外 |
| 执行复核 | 大资金策略没有整手/分钟容量或成交覆盖复核 |

### 晋级标准

| 决策 | 标准 |
|---|---|
| `promote_challenger` | 年化高于 baseline，或年化不低于 baseline - 1pct 且显著改善风险；Calmar 高于 baseline 至少 5% 或绝对提升 >= 0.20；Sortino 不低于 baseline；最大回撤不比 baseline 恶化超过 1pct 且相对恶化不超过 10%；CVaR 5%、worst5d、worst20d 不明显变差；年度/牛熊/成本/TopN/资金规模场景中 Calmar 胜率 >= 60%；容量折损后仍高于 baseline，理论收益保留率 >= 85%。 |
| `keep_shadow` | 年化可低于 baseline，但通常不超过 2pct；至少明显改善最大回撤、CVaR、worst5d/worst20d、恢复天数或成交覆盖之一；能解释坏持仓、拥挤、成交失败、极端状态或行业/事件风险；不改变默认持仓，只作为 warning、半权 shadow、审计字段或下一轮 mutation parent。 |
| `reject` | 只在单一年份/短窗口有效；理论回测胜出但执行后输给 baseline；年化提升但尾部风险或回撤恢复明显变差；参数邻域脆弱；增量只是重复已有 liquidity/tail-risk 暴露。 |

### 训练 / 验证 / 测试

策略研究必须区分样本角色：

| 阶段 | 用途 | 是否允许调参 |
|---|---|---|
| Discovery / 训练集 | 找因子方向、调阈值、生成候选 | 可以，但必须记录参数搜索范围 |
| Validation / 验证集 | 选择少数候选，做参数邻域和 robustness | 可以有限选择，不能反复回看 |
| Test / 测试集 | 最终样本外验收 | 不允许调参，只能一次性报告 |

默认低成本方法是 walk-forward：用过去一段训练，下一段测试，然后向前滚动。

更严格的方法是 **CPCV（Combinatorial Purged Cross-Validation，组合式净化交叉验证）**：

1. 把时间序列切成 N 个连续区块。
2. 每次选择其中 k 个区块作为测试集，剩余区块作为训练集。
3. 遍历多种测试区块组合，得到多条样本外路径。
4. 训练集与测试集之间做 `purge`，删除与测试标签窗口重叠的训练样本。
5. 测试集前后做 `embargo`，留出缓冲期，避免持仓期或事件影响穿透。

CPCV 报告不只看单条样本外曲线，而看 OOS Calmar、Sortino、年化、最大回撤和 win rate 的分布。建议要求 OOS path 打赢 baseline 的比例 >= 60%，且 OOS Calmar 中位数高于 baseline。

### Quant harness engineer

`harness engineer` 在 Vortex 里定义为 **量化评测线束工程师**。它不是只写因子的人，而是负责把“怎么测策略”工程化的人。

| 职责 | 落地方式 |
|---|---|
| 设计测试集 | 年度、牛熊、事件密度、行业、流动性、资金规模切分 |
| 防数据泄漏 | PIT、purge、embargo、未来数据检查 |
| 建指标协议 | Calmar、Sortino、CVaR、worst5d、执行折损、OOS win rate |
| 自动化回归 | 新因子不能破坏已有 baseline / paper challenger |
| 实验可复现 | 固定 artifact schema、参数、数据版本、随机种子 |
| 失败归因 | 区分 alpha 失败、执行失败、容量失败、过拟合失败 |
| 晋级门禁 | promote / shadow / reject 不靠感觉，而靠协议 |

产品判断：Vortex 后续不仅要做 factor engineering，也要做 quant harness engineering。评测协议和回测线束的重要性不低于因子本身。

## 因子进入策略的标准

新增因子至少满足以下之一，才值得进入策略实验：

1. **候选池排序**：在业绩预告候选池内，能稳定提升 top bucket 后续收益。
2. **坏持仓过滤**：能识别未来显著拖累组合的持仓。
3. **弱市防守**：能在市场变差时减少左尾，同时踏空成本可控。
4. **执行改善**：能提高开盘成交率或降低冲击成本。
5. **低相关组合腿**：与主 alpha 相关低，并在主策略空仓或弱年份贡献收益。

## 当前因子的策略定位

| 因子 | 当前定位 | 是否进 live 默认 |
|---|---|---|
| 业绩预告漂移 | 主 alpha | 是，paper/shadow 入口 |
| 集合竞价反向信号 | 执行风险 shadow | 否 |
| 动量101残差 | 持仓/候选池 shadow 降风险 | 否 |
| 筹码成本分布宽度 | 中周期候选池排序线索 | 否，待组合测试 |
| 峰岭谷 PRV | 分钟状态/择时线索 | 否，等全 A 数据和口径复核 |
| 质量/红利/自由现金流 | 防守腿线索 | 否，待组合腿测试 |
| 资金流拥挤反转 | 排序/风险线索 | 否，需全 A 精确复验 |
