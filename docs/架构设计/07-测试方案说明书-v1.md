## 1. 测试原则与分层策略

### 1.1 测试目标

| 目标 | 优先级 | 验收标准 |
|------|--------|---------|
| **确定性可复现** | 🔴 P0 | 同输入重复 N 次，关键输出 bit-identical |
| **功能正确性** | 🔴 P0 | 每个产品功能至少 3 个测试用例（正常/边界/异常） |
| **Fail-Closed 保证** | 🔴 P0 | 质量门禁/风控失败 → 零下游产出 |
| **幂等性** | 🟡 P1 | 标记为幂等的操作重复执行结果一致 |
| **性能基线** | 🟡 P1 | 关键操作不超过约定时间 |
| **跨环境一致** | 🟢 P2 | macOS 与 Linux 结果一致 |

### 1.2 测试金字塔

```
          ┌─────────┐
          │  E2E    │  ~10%  全流程场景（日常运行/故障恢复/新因子生命周期）
         ─┤  Tests  ├─
        ┌─┴─────────┴─┐
        │ Integration  │  ~20%  跨模块交互、DB 读写、外部 Mock
       ─┤    Tests     ├─
      ┌─┴──────────────┴─┐
      │    Unit Tests     │  ~70%  纯函数、计算逻辑、规则引擎、状态机
      └──────────────────┘

      + 横切面：确定性专项测试（贯穿所有层）
```

## 2. 测试设计方法学

### 2.1 基础方法

| 方法 | 适用场景 | 在本系统的应用 |
|------|---------|--------------|
| **等价类划分** | 输入分合法/非法类 | Profile 合法/非法、snapshot 存在/不存在、因子已注册/未注册 |
| **边界值分析** | 数值/日期边界 | 持仓数=0/1/max、权重=0/max_single、日期=最早/最新/非交易日 |
| **判定表** | 多条件组合 | 风控 4 规则 × 通过/不通过 = 16 组合；质量 38 规则组合 |
| **状态迁移** | 状态机 | 订单状态 7 态 × 合法/非法转移；数据管线 running→success/failed |
| **Pairwise** | 参数组合爆炸 | 市场×频率×引擎×成本模型 的组合覆盖 |
| **错误推测** | 经验驱动 | 缺数据、重复日期、NaN 列、ann_date 冲突、时区错配 |

### 2.2 确定性专用方法

| 方法 | 说明 | 应用 |
|------|------|------|
| **Golden Master** | 固定输入→输出 hash 不变 | 回测结果、因子评测结果、组合编译结果 |
| **Property-based** | 验证数学不变量 | 权重和=1、|IC|≤1、无负权重、输入行数=输出行数 |
| **Metamorphic** | 语义不变变换→结果不变 | 因子加常数→RankIC 不变；权重等比放大→组合不变 |
| **Differential** | 不同执行条件→相同结果 | 并发=1 vs 3 一致；串行 vs 并行一致 |
| **Replay** | 冻结输入重放 | 录制 Tushare 响应→回放→逐字节对比 |
| **N-Repeat** | 重复 N 次一致 | 回测 10 次、评测 5 次 → hash 一致 |

## 9. 确定性专项测试

> 本章是整个测试方案的核心——直接对应用户痛点"同条件回测结果不一致"。

### 9.1 确定性威胁模型

| 威胁来源 | 具体表现 | 防御机制 | 测试验证 |
|---------|---------|---------|---------|
| **DataFrame 行序** | groupby/merge 后行序不确定 | 所有输出 canonical sort | DT-SORT-* |
| **排序 tie-break** | 同分股票排序不稳定 | 二级键 symbol ASC | DT-S-005 |
| **浮点累积** | 长时间序列精度漂移 | 6 位小数容忍度 | DT-S-007 |
| **随机数** | random-dates 每次不同 | seed = sha256(profile_hash) | DT-S-003 |
| **并发竞争** | 多 worker 共享状态 | 独立 workdir + cache namespace | DT-CONC-* |
| **Qlib 全局状态** | Qlib provider 单例 | 每 worker 独立初始化 | DT-CONC-002 |
| **时间依赖** | 代码中使用 datetime.now() | 冻结时间 (freezegun) | DT-TIME-* |
| **输入引用漂移** | latest 别名指向变化 | 提交时解析为具体 ID | DT-REF-* |
| **依赖版本** | numpy/pandas 升级改变行为 | 固定版本 + dependency_versions 审计 | DT-DEP-* |
| **PYTHONHASHSEED** | dict/set 迭代顺序不确定 | 统一设置 `PYTHONHASHSEED=0` | DT-ENV-001 |
| **BLAS 多线程** | 浮点归约顺序不确定 | `OMP_NUM_THREADS=1` 等 | DT-ENV-002 |
| **文件枚举顺序** | glob/listdir 不确定 | 结果统一 `sorted()` | DT-ENV-003 |
| **序列化顺序** | JSON key 顺序影响 hash | `json.dumps(sort_keys=True)` | DT-ENV-004 |

**确定性验收双层标准**：
- **持久化层**（落盘 DataFrame、指标、运行记录）：canonical rounding 后 hash 完全一致（bit-identical）
- **展示层**（HTML 报告、CLI 输出、通知消息）：允许 6 位小数容忍（`|a-b| < 1e-6`）
- 所有"是否一致"的判断以持久化层为准；展示层微小差异不构成 Bug
