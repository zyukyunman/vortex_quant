## 1. 跨域统一约束

本章定义所有域必须遵守的全局规则。任何域的实现如果违反本章约定，视为 **阻断级 Bug**。

### 1.1 确定性契约（Determinism Contract）

#### 1.1.1 run_id 与 reproducibility_key 分离

系统中存在两种标识，职责不同，严禁混淆：

| 概念 | 格式 | 用途 | 是否确定性 |
|------|------|------|-----------|
| **run_id** | `{domain}_{YYYYMMDD}_{HHMMSS}_{random4}` | 唯一标识一次执行实例 | ❌ 含随机后缀 |
| **reproducibility_key** | `sha256(profile_hash + snapshot_id + code_hash + dep_versions)` | 标识"相同输入" | ✅ 必须确定性 |

**规则**：
- 回测、评测、编译等操作在**提交时**必须将所有引用解析为具体 ID，禁止 `latest` / `latest_success` 等别名进入计算逻辑
- 解析后的具体 ID 记录在运行记录的 `resolved_refs` 字段中
- 去重、比较、回归测试一律使用 `reproducibility_key`，不使用 `run_id`

#### 1.1.2 输出排序规则（Canonical Sort）

所有输出在落盘前必须做**主键排序**，禁止依赖 DataFrame 当前行序：

```python
# ✅ 正确
df = df.sort_values(["date", "symbol"]).reset_index(drop=True)

# ❌ 错误 — 依赖 groupby/merge 后的行序
df = df.groupby("sector").apply(fn)  # 行序不确定
```

#### 1.1.3 排序 Tie-Breaking

所有涉及排名/排序的操作必须定义**二级键**：

| 场景 | 主键 | 二级键 | 示例 |
|------|------|--------|------|
| 因子排名 | score DESC | symbol ASC | 分组时 score 相同的股票有确定性顺序 |
| 组合权重 | weight DESC | symbol ASC | 选 Top-N 时确定性截断 |
| IC 排名 | ic DESC | factor_name ASC | 因子比较时确定性排序 |

### 1.2 舍入规则（Rounding Convention）

全系统统一的精度与舍入标准：

| 数据类型 | 精度 | 舍入方式 | 依据 |
|---------|------|---------|------|
| 股价 | 2 位小数 | 四舍五入 | A 股最小变动单位 0.01 元 |
| 权重 | 4 位小数 | 四舍五入 | 精度需求：1.0000 表示 100% |
| IC / RankIC | 6 位小数 | 四舍五入 | 统计量需高精度 |
| Sharpe | 4 位小数 | 四舍五入 | 行业惯例 |
| 收益率 | 6 位小数 | 四舍五入 | 累计时精度要求高 |
| 手数 | 整数 | **向下取整** | A 股 100 股/手，floor(shares/100)*100 |
| 佣金 | 2 位小数 | **向上取整** | ceiling 到分，保护模拟精度 |
| 滑点 (bps) | 1 位小数 | 四舍五入 | (fill-limit)/limit × 10000 |
| 百分位排名 | 2 位小数 | 四舍五入 | 0.00-100.00 |

**关键不变量**：
- 组合权重和：`|sum(weights) - 1.0| < 1e-4`
- 所有权重非负：`all(weights >= 0)`

### 1.3 时间与时区

#### 1.3.1 系统唯一时区

```
系统时区 = Asia/Shanghai (UTC+8)
所有 datetime 字段存储为带时区的 UTC+8
跨时区场景（如港股、美股）：转换为 UTC+8 后存储

DDL 规范：所有 TIMESTAMP 字段存储 ISO8601 with offset 格式（如 '2026-04-04T22:30:00+08:00'）
         禁止依赖数据库默认的 CURRENT_TIMESTAMP（可能不带时区）
         写入时由应用层统一生成 UTC+8 带 offset 的时间字符串

多市场兼容说明：
  当前 MVP 仅支持 A 股（Asia/Shanghai）
  未来扩展时，时区和交易时段将下沉到 market/calendar 配置（由 Profile 的 preset 提供）
  核心时间逻辑（as_of 判定、PIT 对齐）接受 timezone 参数，不硬编码 UTC+8
  v1 实现中 timezone 参数默认值为 Asia/Shanghai，其他市场待 v2 支持
```

#### 1.3.2 Snapshot as_of 判定

```
规则：当日 22:00 后生成的 snapshot 归属当日 as_of
      22:00 前生成的归属前一个交易日

示例：
  2026-04-04 22:30 生成 → as_of = 2026-04-04
  2026-04-04 21:50 生成 → as_of = 2026-04-03（前一个交易日）
```

#### 1.3.3 交易日历

- 由 `DataProvider.fetch_calendar()` 提供，缓存在 `trading_calendar` 表
- 非交易日：跳过，不填充
- 半日市（如春节前一天）：正常处理，仅数据量少
- 分钟频数据边界：09:30-11:30, 13:00-15:00，非交易时段数据丢弃

#### 1.3.4 PIT（Point-In-Time）判定规则

```
规则 1：财报数据以 ann_date（公告日期）为准，NOT report_date
规则 2：effective_from = ann_date 的下一个交易日 09:30
规则 3：若 ann_date 在交易时段内（09:30-15:00），当日 15:00 后生效
规则 4：快报、预告、正式报各自独立 PIT，互不覆盖
规则 5：ann_date 为非交易日 → effective_from = 下一个交易日 09:30

示例：
  ann_date = 2026-03-15（周日）→ effective_from = 2026-03-16 09:30（周一）
  ann_date = 2026-03-14 14:00（盘中）→ effective_from = 2026-03-14 15:00
```

### 1.4 并发与隔离

```
默认 max_concurrent_tasks = 3（可通过 Profile 配置）

隔离规则：
  1. 每个 run 独立工作目录：{workspace}/runs/{run_id}/
  2. 因子缓存以 (factor_name, snapshot, code_hash) 为键，天然隔离
  3. 每个 worker 独立初始化 Qlib provider（避免全局状态污染）
  4. 临时文件：{workspace}/tmp/{run_id}/，完成后清理
  5. DB 写操作使用行级锁或 UPSERT 语义，避免并发冲突
```

#### 1.5.3 失败策略

| 场景 | 策略 | 说明 |
|------|------|------|
| 质量门禁失败 | **Fail-Closed** | 阻断下游，不降级 |
| 风控规则失败 | **Fail-Closed** | 整批阻断，零订单提交 |
| 网络/外部 API | **自动重试** | 3 次，指数退避 5min→15min→30min |
| AI 辅助不可用 | **降级** | 回退为"未分类"，不阻断流程 |
| 通知发送失败 | **降级+重试** | 重试 3 次后降级消息，连续 3 次失败暂停订阅 |

```
规则：
  1. 每张 DB 表有 schema_version 字段（INTEGER DEFAULT 1）
  2. 迁移脚本存放在 vortex/migrations/{table_name}_v{N}_to_v{N+1}.py
```

### 1.8 环境确定性契约

除了数据和代码层面的确定性保证，运行环境本身也可能引入不可复现性。以下环境因素必须固定：

| 威胁来源 | 影响 | 防御措施 |
|---------|------|---------|
| `PYTHONHASHSEED` | dict/set 迭代顺序不确定 | 生产/测试环境统一设置 `PYTHONHASHSEED=0` |
| BLAS/OpenMP/MKL 多线程 | 浮点归约顺序不确定 | 设置 `OMP_NUM_THREADS=1`, `MKL_NUM_THREADS=1`, `OPENBLAS_NUM_THREADS=1` |
| 文件枚举顺序 | `glob()` / `os.listdir()` 返回顺序不确定 | 所有文件列表操作后必须 `sorted()` |
| JSON/YAML 序列化 | key 顺序不确定影响 hash | 统一使用 `json.dumps(sort_keys=True)` 或 canonical YAML |
| 随机种子 | NumPy/Pandas 采样不确定 | 所有随机操作显式传入 `random_state=42`（可通过 Profile 配置） |

**启动检查**：系统启动时由 `EnvironmentChecker` 验证以上环境变量，不满足时打印 warning 并记录到审计日志。

### 1.9 确定性验收双层标准

系统的确定性验收分为两层，不可混淆：

| 层级 | 适用对象 | 标准 | 用途 |
|------|---------|------|------|
| **持久化层** | 落盘的 DataFrame、指标、运行记录 | canonical rounding 后 hash 一致（bit-identical） | 归档去重、golden master、回归测试 |
| **展示层** | HTML 报告、CLI 输出、通知消息 | 允许 6 位小数容忍（`|a-b| < 1e-6`） | 人类阅读，不参与比较 |

规则：所有"是否一致"的判断以**持久化层**为准；展示层的微小差异不构成 Bug。

### 2.2 核心接口契约

```python
from typing import Protocol
import pandas as pd
from datetime import date


class DataProvider(Protocol):
    """数据源接口 — 所有 provider 必须实现"""

    @property
    def name(self) -> str:
        """provider 标识，如 'tushare'"""
        ...

    @property
    def supported_markets(self) -> list[str]:
        """支持的市场列表，如 ['cn_stock']"""
        ...

    def fetch_instruments(self, market: str) -> pd.DataFrame:
        """获取标的列表。
        返回列：symbol, name, list_date, delist_date, industry, market_cap
        排序：symbol ASC
        异常：DataError(DATA_PROVIDER_FETCH_FAILED)
        """
        ...

    def fetch_calendar(self, market: str, start: date, end: date) -> list[date]:
        """获取交易日历。返回已排序的交易日列表。
        不变量：返回日期 ⊆ [start, end]
        """
        ...

    def fetch_bars(
        self, market: str, symbols: list[str],
        freq: str, start: date, end: date
    ) -> pd.DataFrame:
        """获取行情数据。
        返回列：symbol, date, open, high, low, close, volume, amount
        不变量：返回数据的日期 ⊆ [start, end]
        不变量：返回的 symbol ⊆ 请求的 symbols
        排序：date ASC, symbol ASC
        异常：DataError(DATA_PROVIDER_FETCH_FAILED)
        """
        ...

    def fetch_fundamental(
        self, market: str, symbols: list[str],
        fields: list[str], start: date, end: date
    ) -> pd.DataFrame:
        """获取基本面数据。
        返回列：symbol, ann_date, report_date, ...fields
        PIT 要求：必须包含 ann_date 字段
        排序：ann_date ASC, symbol ASC
        """
        ...

    def fetch_events(
        self, market: str, symbols: list[str],
        start: date, end: date
    ) -> pd.DataFrame:
        """获取事件数据（分红、配股、停牌等）。
        排序：date ASC, symbol ASC
        """
        ...


class StorageBackend(Protocol):
    """存储后端接口"""

    def upsert(self, dataset: str, df: pd.DataFrame, partition: dict) -> int:
        """写入/覆盖数据。返回写入行数。
        幂等：相同 partition 覆盖写，不产生重复
        不变量：写入后立即可读
        """
        ...

    def read(
        self, dataset: str, filters: dict | None = None,
        columns: list[str] | None = None
    ) -> pd.DataFrame:
        """读取数据。filters 为 DuckDB WHERE 条件字典。
        返回：按 date ASC, symbol ASC 排序的 DataFrame
        """
        ...

    def list_partitions(self, dataset: str) -> list[str]:
        """列出已有分区。返回：排序后的分区路径列表"""
        ...

    def snapshot(self, profile: str, as_of: date) -> str:
        """发布快照。返回 snapshot_id。
        前置条件：质量检查已通过
        不变量：snapshot_id 一旦生成不可变更
        约束：同 profile + as_of 只允许一个有效快照
        """
        ...
```

```python
    def bootstrap(self, profile: ResolvedProfile, dry_run: bool = False) -> RunReport:

        ...

        """
        ...

        """修复：仅重跑 run_id 对应的 failed_units。

        ...

        ...
```

### 2.5 控制面 DDL

```sql
-- ========================================
-- 数据域控制面表
-- ========================================

-- 同步清单：每个 profile/dataset/date 的同步状态
CREATE TABLE sync_manifest (
    profile         TEXT    NOT NULL,
    dataset         TEXT    NOT NULL,
    trade_date      DATE    NOT NULL,
    status          TEXT    NOT NULL DEFAULT 'pending',
    -- pending | syncing | success | failed
    run_id          TEXT,
    chunk_id        TEXT,
    row_count       INTEGER,
    checksum        TEXT,           -- 数据内容 sha256
    created_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
```

### 3.3 因子评测指标计算口径

> 这些口径是确定性测试的 Golden Master 基准，任何修改需同步更新测试。

```
═══════════════════════════════════════════
IC 计算
═══════════════════════════════════════════
IC_t(period) = SpearmanCorr(factor_t, return_{t → t+period})
  period ∈ {1d, 5d, 20d}，各自独立计算
  ties 处理方式：average rank（与 scipy.stats.spearmanr 一致）
  每个截面至少 30 个有效样本，否则该截面 IC = NaN

RankIC_t(period) = PearsonCorr(rank(factor_t), rank(return_{t → t+period}))

ICIR = mean(IC_series) / std(IC_series)
  分母为零时 ICIR = 0
  年化 ICIR = ICIR × √(交易日数 / period)

═══════════════════════════════════════════
IC 衰减
═══════════════════════════════════════════
对 period ∈ [1, 2, 3, ..., 20]：
  IC_mean(period) = mean(IC_t(period) for all t)
衰减速率 = OLS slope(period, IC_mean)
  负斜率 = 衰减快（短期因子）
  接近零 = 稳定因子

═══════════════════════════════════════════
分组收益
═══════════════════════════════════════════
每期按因子值排序 → 等分 N 组（N=5 或 10）
  排序规则：factor_value DESC, symbol ASC
各组等权持仓 → 计算该期组收益
多空收益 = 第1组收益 - 第N组收益
多空 Sharpe = mean(多空日收益) / std(多空日收益) × √252

═══════════════════════════════════════════
容量估算
═══════════════════════════════════════════
按市值等分 5 档 → 各档分别计算 IC_mean
容量门槛 = IC_mean > 0.03 的最小市值分档下界
估算容量（元）= 门槛市值 × 持仓数 × 日均换手率 × 参与率(10%)

═══════════════════════════════════════════
Regime 分类
═══════════════════════════════════════════
基准指数：默认 000300.SH（可配置）
Bull:   MA120 > 前值 且 slope(MA120, 20d) > 0
Bear:   MA120 < 前值 且 slope(MA120, 20d) < 0
Choppy: 其他
slope = OLS 斜率对 20 个交易日的 MA120 值
```

### 5.4 订单状态机

```
                    ┌──────────────────────────────────────┐
                    │           Order State Machine         │
                    └──────────────────────────────────────┘

    ┌─────────┐  submit_ok   ┌───────────┐  partial   ┌──────────────┐
    │ pending │─────────────→│ submitted │──────────→│ partial_fill │
    └─────────┘              └───────────┘            └──────────────┘
                                  │                         │
                                  │ full_fill               │ full_fill
                                  ▼                         ▼
                             ┌──────────┐             ┌──────────┐
                             │  filled  │             │  filled  │
                             └──────────┘             └──────────┘

                             ┌──────────┐
                  rejected  →│ rejected │  （网关拒绝：资金不足/涨跌停等）
                             └──────────┘

                             ┌──────────┐   retry    ┌─────────┐
                  timeout   →│ timeout  │───────────→│ pending │ （需重新 pre_trade_check）
                             └──────────┘            └─────────┘

                             ┌───────────┐
                  cancel    →│ cancelled │  （用户取消 or intra_trade 风控触发）
                             └───────────┘

合法状态转移表：
  pending     → submitted | cancelled
  submitted   → partial_fill | filled | rejected | timeout | cancelled
  partial_fill → filled | timeout | cancelled
  timeout     → pending（retry）| cancelled
  filled      → (终态)
  rejected    → (终态)
  cancelled   → (终态)

非法转移 → 抛出 TRADE_ILLEGAL_STATE_TRANSITION
```

| `watch.report.ready` | 🟢 info | 定期关注报告就绪 |
| `watch.provider.failed` | 🟡 warning | Watch provider 执行失败 |

**默认通知行为**（产品 07-v2 §5.1）：
- `critical` 和 `warning` 事件：**默认开启推送**
- `info` 事件（如 `data.sync.success`、`research.eval.complete`）：**默认静默**，用户需显式开启
- 日常自动运行流程中的 `info` 通知仅写入 `notification_log`，不实际推送（除非用户配置了 `info` 级别路由）

补充事件类型（产品 07-v2 新增）：

| 事件类型 | 严重级别 | 触发场景 |
|---------|---------|---------|
| `data.partial_update_failed` | 🟡 warning | 部分品种更新失败 |
| `research.factor.grade_a` | 🟡 warning | 发现 A 级因子（需关注） |
| `strategy.quality.cleanup_suggested` | 🟡 warning | D/F 级回测积压，建议清理 |
| `watch.consecutive_failures` | 🟡 warning | Watch provider 连续失败 ≥ 3 次 |

### 7.2 Profile 解析流程

```
用户 YAML 文件
  │
  ▼
ProfileLoader.load(path) → RawProfile
  │
  ▼
ProfileValidator.validate(raw) → 校验通过 / ConfigError
  │
  ▼
ProfileResolver.resolve(raw)
  ├── PresetRegistry.get(preset_name) → PresetProfile
  ├── PackRegistry.get(pack_name) × N → PackConfig
  ├── ProfileMerger.merge(raw, preset, packs) → 合并

  │

  - preset 展开字段（provider, datasets, storage 等）
  - pack 展开字段（quality_rules, pit_rules, risk_rules 等）

  - reproducibility_key
```
