## 一、策略设计

### 1.1 投资逻辑

本策略基于两层分离的思路：

1. **选什么（Selection）**：用低频价值因子做股票池筛选——自由现金流收益率（FCF Yield）和低波动率（Low Volatility）。这两个因子在学术上是最强的选股因子组合（AQR QMJ 框架 + JKP 跨国因子分析均证实）。
2. **何时调（Timing）**：选完股后不立即调仓，而是用动量信号决定买卖时机——股池（选股结果 ∪ 当前持仓）中，买入动量转正的、卖出动量转负的。

这一设计的核心假设：低频价值选股中，立即调仓与延后调仓对组合表现影响不大（因为价值因子的 alpha 衰减慢），但用动量择时可以避免买在下跌途中、减少不必要的回撤。

> **学术支撑**：AQR "Value and Momentum Everywhere" 证实 value 和 momentum 负相关（correlation ≈ -0.5），组合使用显著降低波动率。JKP 研究确认在 A 股市场（散户主导），动量因子尤其有效。

### 1.2 因子定义

#### 选股因子（Research 域计算，季度频率）

| 因子 | 定义 | 数据口径 | 学术依据 |
|------|------|---------|---------|
| **fcf_yield** | 自由现金流 / 企业价值（EV） | TTM（trailing twelve months，滚动四季度合计）。自由现金流 = 经营活动现金流净额 - 资本支出。企业价值 = 总市值 + 有息负债 - 货币资金 | 980092 编制方案核心指标；JKP 中 Ocp（Operating CF / Price）T-stat = 8.36 |
| **low_vol** | 过去 252 个交易日日收益率的年化标准差的倒数 | 日频收盘价对数收益率，年化 = sigma_daily * sqrt(252) | AQR BAB（Betting Against Beta）因子；JKP 中低波因子在新兴市场防御力显著 |

#### 质量过滤条件（Universe 阶段执行）

参考 980092 编制方案的筛选规则：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 上市时间 | >= 120 个交易日 | 排除次新股 |
| 日均成交额 | 最近 120 日均值 >= 500 万 | 流动性门槛 |
| ST / *ST | 排除 | -- |
| 行业 | 排除金融、房地产 | 980092 规则 |
| ROE 稳定性 | 最近 12 季度 ROE 标准差排名后 10% 剔除 | 980092 规则 |
| 近一年 FCF > 0 且经营活动现金流 > 0 | 硬约束 | 剔除现金流为负的公司 |

#### 择时因子（Research 域计算，周频率）

| 因子 | 定义 | 数据口径 | 学术依据 |
|------|------|---------|---------|
| **momentum_20d** | 过去 20 个交易日累计收益率 | 日频收盘价复权 | 短期横截面动量；JKP 中 momentum 在中国 A 股市场 T-stat 排名前三 |

> **关于峰岭谷因子**：因子分析报告中 ridge_minutes 的 IC_5d = 0.308，预测力极强，但依赖分钟级数据。在 MVP 阶段使用日频价格动量；后续可引入峰岭谷因子作为增强。

### 1.3 策略参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 选股频率 | 季度（3/6/9/12 月） | 与 980092 一致 |
| 择时频率 | 周 | 每周执行一次 compile |
| 股池规模 | 50 只 | 从 FCF Yield x Low Vol 综合排名前 50 |
| 目标持仓 | 20~30 只 | 动量排名后截取 |
| 个股权重上限 | 5% | 等权基础上的硬约束 |
| 行业权重上限 | 20% | 参考 980092 |

| 基准 | 沪深300（000300.SH） | -- |

### 1.4 Pipeline 映射

```
用户逻辑                    Pipeline 阶段              具体实现
---                        ---                       ---
全市场 + 质量过滤          -> Universe                  base="全A", filters=[listing_days, liquidity, exclude_st, exclude_industry, roe_stability, positive_fcf]
选股结果 U 当前持仓        -> Universe                  merge_current_holdings=true, symbols = filtered U current_holdings
FCF + LowVol 综合排名      -> Alpha stage 1 (cascading) signal: sig_value_composite, action: filter_top_n, top_n: 50, preserve_current_holdings: true
动量排序                   -> Alpha stage 2 (cascading) signal: sig_momentum_20d, action: score_filter
按动量分数赋权              -> Portfolio Construction    scheme: signal_weighted
个股/行业约束              -> Risk                      max_weight: 0.05, sector_limit: 0.20, min_positions: 20
```

**关键扩展点（已在 04 §3.3 和 §3.7 中定义）**：

1. **`current_holdings` 注入 PipelineContext**：compile 模式下通过 `--current-holdings` 注入；backtest 模式下由 BacktestEngine 自动填入上一期持仓。此字段对 Universe 和 Alpha 阶段均可见。
2. **Universe `merge_current_holdings`**：Universe 输出 = 过滤后标的 ∪ current_holdings（确保当前持仓不被质量过滤直接剔除）。
3. **Alpha `filter_top_n` 的 `preserve_current_holdings` 参数**：输出 = top-N 标的 ∪ current_holdings 中仍在候选池的标的。这使得跌出价值排名的当前持仓不会被立即卖出，而是留到 momentum_timing 阶段由动量信号决定去留。

#### 2.2 因子代码示例

```python
# factors/fcf_yield.py
"""
自由现金流收益率因子。
FCF Yield = Free Cash Flow (TTM) / Enterprise Value
其中：
  FCF = 经营活动现金流净额 - 资本支出（购建固定资产等支出）
  EV  = 总市值 + 有息负债 - 货币资金
数据口径：TTM（滚动四季度合计），PIT 对齐。
"""
from vortex.research import FactorBase

class FcfYieldFactor(FactorBase):
    name = "fcf_yield"
    datasets = ["cashflow", "balancesheet", "daily_basic"]
    frequency = "quarterly"           # 因子值按季度更新

    def compute(self, data, as_of):
        cf = data["cashflow"]         # PIT 对齐后的现金流
        bs = data["balancesheet"]     # PIT 对齐后的资产负债表
        mkt = data["daily_basic"]     # 市值数据

        # TTM 合计：最近四个季度的经营活动现金流和资本支出
        operating_cf_ttm = cf.groupby("symbol")["n_cashflow_act"].rolling(4).sum()
        capex_ttm = cf.groupby("symbol")["c_pay_acq_const_fiolta"].rolling(4).sum()
        fcf_ttm = operating_cf_ttm - capex_ttm

        # 企业价值 = 总市值 + 有息负债 - 货币资金
        # 有息负债 = 短期借款 + 长期借款 + 应付债券（非全部负债）
        total_mv = mkt["total_mv"]
        interest_debt = bs.get("st_borr", 0) + bs.get("lt_borr", 0) + bs.get("bond_payable", 0)
        cash = bs["money_cap"]
        ev = total_mv + interest_debt - cash
        ev = ev.replace(0, np.nan)    # EV 为零时标记为无效

        # FCF Yield = FCF / EV（越高越好）
        fcf_yield = fcf_ttm / ev
        return fcf_yield
```

```python
# factors/low_vol_inv.py
"""
低波动因子（倒数形式）。
Low Vol Inv = 1 / annualized_volatility(252d)
值越高表示波动越低（越好）。
"""
import numpy as np
from vortex.research import FactorBase

class LowVolInvFactor(FactorBase):
    name = "low_vol_inv"
    datasets = ["bar_1d"]
    frequency = "daily"

    def compute(self, data, as_of):
        prices = data["bar_1d"]["close"]
        log_ret = np.log(prices / prices.shift(1))
        vol_252 = log_ret.rolling(252).std() * np.sqrt(252)
        # clip 防止除零：年化波动率低于 1% 视为异常（停牌或数据缺失）
        return 1.0 / vol_252.clip(lower=0.01)
```

```python
# factors/momentum_20d.py
"""
20 日价格动量因子。
Momentum_20d = close(T) / close(T-20) - 1
"""
from vortex.research import FactorBase

class Momentum20dFactor(FactorBase):
    name = "momentum_20d"
    datasets = ["bar_1d"]
    frequency = "daily"

    def compute(self, data, as_of):
        close = data["bar_1d"]["close"]
        return close / close.shift(20) - 1
```

```python
# factors/value_composite.py
"""
价值复合因子：FCF Yield (60%) + Low Vol Inv (40%) 加权融合。
在因子代码层面完成融合，Research 域按单因子流程评测和发布。
"""
import numpy as np
from vortex.research import FactorBase

class ValueCompositeFactor(FactorBase):
    name = "value_composite"
    datasets = ["cashflow", "balancesheet", "daily_basic", "bar_1d"]
    frequency = "quarterly"

    def compute(self, data, as_of):
        # FCF Yield 计算（同 fcf_yield.py）
        cf = data["cashflow"]
        bs = data["balancesheet"]
        mkt = data["daily_basic"]
        operating_cf_ttm = cf.groupby("symbol")["n_cashflow_act"].rolling(4).sum()
        capex_ttm = cf.groupby("symbol")["c_pay_acq_const_fiolta"].rolling(4).sum()
        fcf_ttm = operating_cf_ttm - capex_ttm
        total_mv = mkt["total_mv"]
        # 有息负债 = 短期借款 + 长期借款 + 应付债券（非全部负债）
        interest_debt = bs.get("st_borr", 0) + bs.get("lt_borr", 0) + bs.get("bond_payable", 0)
        cash = bs["money_cap"]
        ev = total_mv + interest_debt - cash
        fcf_yield = fcf_ttm / ev.replace(0, np.nan)

        # Low Vol Inv 计算（同 low_vol_inv.py）
        prices = data["bar_1d"]["close"]
        log_ret = np.log(prices / prices.shift(1))
        vol_252 = log_ret.rolling(252).std() * np.sqrt(252)
        low_vol_inv = 1.0 / vol_252.clip(lower=0.01)  # 防止除零

        # 横截面标准化后加权融合
        fcf_z = (fcf_yield - fcf_yield.mean()) / fcf_yield.std().clip(lower=1e-8)
        vol_z = (low_vol_inv - low_vol_inv.mean()) / low_vol_inv.std().clip(lower=1e-8)
        return 0.6 * fcf_z + 0.4 * vol_z
```

### Phase 3：策略回测（Strategy 域）

#### 3.1 编写 Strategy Profile

```yaml
# config/profiles/fcf_momentum.yaml
profile_type: strategy

# 管线配置
universe:
  base: "全A"                         # 全市场（非指数成分限定）
  filters:
    - listing_days_min: 120
    - liquidity_min: 5000000
    - exclude_st: true
    - exclude_industry: ["银行", "非银金融", "房地产"]
    - roe_stability_min_pct: 10       # 剔除 ROE 稳定性后 10%
    - positive_fcf: true              # 近一年 FCF > 0
  merge_current_holdings: true        # 核心：当前持仓合并到候选池

alpha:
  mode: cascading
  stages:
    - name: value_selection
      signal_id: sig_value_composite  # FCF + Low Vol 融合信号
      action: filter_top_n
      params: { top_n: 50, preserve_current_holdings: true }  # 当前持仓不因跌出 top-50 被剔除
    - name: momentum_timing
      signal_id: sig_momentum_20d     # 20 日动量信号
      action: score_filter
      params: { threshold: 0.0 }      # 动量 > 0 才考虑买入

portfolio:
  scheme: signal_weighted             # 按动量分数赋权
  normalize: true                     # 权重归一化

risk:
  constraints:
    - max_weight: 0.05                # 个股 <= 5%
    - min_positions: 20               # 至少 20 只
    - max_positions: 30               # 最多 30 只
    - sector_limit: 0.20              # 行业 <= 20%
  runtime_policies:
    - drawdown_stop: -0.15            # 组合回撤超 15% 暂停（仅回测生效）

# 回测配置
backtest:
  engine: backtrader                  # 第一版用 Backtrader
  start: "2015-01-01"
  end: "latest"
  benchmark: "000300.SH"
  initial_capital: 1000000
  commission: 0.0003                  # 万三
  slippage: 0.001                     # 0.1%
  rebalance_frequency: weekly         # 每周执行 compile
```

## 四、验收标准

本实例策略开发完成的验收标准：

### 4.1 端到端通过

从 `vortex data bootstrap` 到 `vortex trade reconcile`，全流程无人工干预地跑通。

### 4.2 数值验收

| 检查项 | 标准 |
|--------|------|
| 数据完整性 | 2014-01-01 至今，bar_1d 行数 >= 预期的 95% |
| 因子覆盖率 | fcf_yield 覆盖率 >= 85%（剔除金融/地产后） |
| IC 方向 | fcf_yield IC_mean > 0（正向有效） |
| 回测收益 | 全样本年化 > 基准年化（策略有超额） |
| 回测鲁棒性 | 随机起点 20 次中 >= 15 次超额为正 |
| 对账偏差 | Paper trade 对账偏差 < 1% |

### 4.3 确定性验收

相同 profile + 相同 snapshot + 相同 code -> bit-identical 回测结果。验证方法：同一参数跑两次，对比 reproducibility_key 和输出文件 hash。

### 4.4 Fail-Closed 验收

| 场景 | 预期行为 |
|------|---------|
| 数据快照质量门禁失败 | 阻断，不发布快照 |
| signal_snapshot 未发布就 compile | 阻断 |
| 约束冲突（如 min_positions 不足） | Pipeline 阻断，不生成 target |
| 涨跌停标的出现在买入计划 | 事前风控拦截 |

### 4.5 核心交易逻辑场景验收

本策略的核心行为是"当前持仓由动量决定去留"，必须通过以下确定性场景测试：

| 场景 | 输入条件 | 预期行为 |
|------|---------|---------|
| 持仓保留 | 当前持仓股 A 不在价值 top-50 内，但 momentum_20d > 0 | A 出现在 target_portfolio 中（未被卖出） |
| 持仓退出 | 当前持仓股 B 不在价值 top-50 内，且 momentum_20d < 0 | B 不在 target_portfolio 中（被卖出） |
| 新进买入 | 非持仓股 C 在价值 top-50 内，且 momentum_20d > 0 | C 出现在 target_portfolio 中（买入） |
| 新进拒绝 | 非持仓股 D 在价值 top-50 内，但 momentum_20d < 0 | D 不在 target_portfolio 中（暂不买入） |
| 约束裁剪 | target 满足前四条但超过 max_positions=30 | Risk 阶段按分数从低到高裁剪至 30 只 |
| 空持仓首次编译 | current_holdings 为空 | 等价于纯 filter_top_n + score_filter，无保留逻辑 |

这些场景应实现为确定性 unit test（构造固定的 signal scores + current_holdings 输入，断言输出）。
