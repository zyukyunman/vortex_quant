# 因子评测与权重筛选知识体系

这篇文档不是解释某一个脚本怎么跑，而是建立一套更稳定的理解框架：

- 因子到底分哪几类
- 每类因子在策略里扮演什么角色
- 因子应该在什么股票池里评测
- 单因子评测和多因子配权怎么衔接

如果你是量化入门者，最重要的不是先记住几个术语，而是先建立一个判断框架：

1. 先分清股票池。
2. 再分清因子角色。
3. 最后才谈评测方法和配权。

## 1. 先纠正一个常见理解

你提出的这条链路：

- 先用估值因子筛一次
- 再上质量和成长因子
- 最后再上动量和波动因子

这个想法不是错的，但它不是通用铁律，更准确的表达应该是：

- 先做基础可交易样本清洗
- 再做硬门槛过滤
- 再做核心打分因子排序
- 最后叠加时机因子和风险因子

也就是说，真正应该先分的不是“估值、质量、成长、动量、波动”这些名字，
而是这些因子在策略里扮演的角色。

一个更稳的理解方式是：

- `过滤因子`：决定哪些股票根本不能进池
- `打分因子`：决定候选股票谁更优
- `时机因子`：决定当前阶段该偏向哪类股票
- `风险因子`：决定组合要不要惩罚某些暴露

同一个类别的因子，在不同策略里角色可以不同。

例如：

- `估值因子` 在高股息策略里可能是核心打分因子
- 在趋势策略里可能根本不用
- `波动率因子` 有时是选股因子，有时只是组合风险惩罚项
- `动量因子` 有时是最终排序核心，有时只是同分条件下的二次筛选

所以更合理的框架不是“固定顺序”，而是“先定角色，再定顺序”。

## 2. 股票池不是只有一个

很多入门者一上来就问“因子到底应该在全市场评测，还是在筛选后的股票池评测”，
其实根本原因是把“股票池”理解成了只有一个。

更合理的做法是至少分 3 层。

### 2.1 原始可交易池

这是最宽的一层，但也不是字面意义上的“所有股票”。

通常至少要先做最基础清洗：

- 去掉 ST
- 去掉上市时间太短的股票
- 去掉长期停牌或明显不可交易样本
- 去掉缺失关键数据的股票

这一层可以叫做：`broad tradable universe`。

它回答的问题是：

- 如果不带太多策略偏见，这个因子在较宽样本上有没有原始预测力。

### 2.2 策略候选池

这是更接近真实策略约束的一层。

例如高股息策略里，候选池可能会额外要求：

- 连续分红年数达到门槛
- 股利支付率在合理区间
- 不属于某些行业
- 现金流、盈利质量达标

这一层回答的问题是：

- 这个因子在“我真正打算投资的那类股票”里有没有用。

### 2.3 最终组合池

这是排序和持仓以后剩下的 Top N 股票。

这一层通常不再做“单因子是否有预测力”的研究，
而是看：

- 组合收益
- 回撤
- 夏普
- 换手
- 风险暴露

也就是说：

- `原始可交易池` 用来测原始因子能力
- `策略候选池` 用来测条件化后的因子能力
- `最终组合池` 用来测整套策略能力

## 3. 因子先按“角色”分类，再按“名称”分类

下面这个抽象比“估值、质量、成长、动量、波动”更重要。

| 角色 | 作用 | 常见输出 | 典型因子 |
| ------ | ------ | --------- | --------- |
| 硬过滤 | 不满足就直接出局 | pass / fail | ST、上市天数、连续分红、财务造假风险 |
| 核心打分 | 对候选股票排序 | 连续分数 | 估值、质量、成长 |
| 时机叠加 | 反映短中期市场偏好 | 加分 / 减分 | 动量、反转、盈利预期修正 |
| 风险惩罚 | 控制组合暴露 | 惩罚项 / 权重上限 | 波动率、流动性、拥挤度、行业偏离 |

这个表背后的意思是：

- 不是所有因子都应该拿来算 IC 后直接配权
- 有些因子应该做门槛，不应该做连续打分
- 有些因子不一定追求“预测收益”，而是追求“降低回撤”或“减少风险”

这是因子研究里最容易混淆的一点。

## 4. 估值、质量、成长、动量、波动分别适合扮演什么角色

### 4.1 估值因子

典型代表：

- `ep`
- `bp`
- `dividend_yield`

常见角色：

- 核心打分因子
- 有时也可以做门槛过滤

适合的问题：

- 哪些股票当前更便宜
- 哪些股票具备再定价空间

估值因子通常更适合：

- 中期到长期 horizon
- 较宽股票池先看原始能力
- 再到策略候选池里看是否仍然有效

### 4.2 质量因子

典型代表：

- `roe_ttm`
- `opcfd`
- `roe_stability`

常见角色：

- 硬过滤
- 核心打分

质量因子最大的特点是：

- 经济学逻辑很强
- 但单独对未来收益的横截面预测力不一定稳定

原因很简单：

- 高质量公司未必便宜
- 如果市场早已充分给高质量公司溢价，未来收益反而可能一般

所以质量因子常常更适合做两件事：

- 先过滤掉明显差公司
- 再在“估值还不算贵”的股票里比较谁质量更好

这也是为什么质量因子很多时候要做条件化评测，而不是只看全市场裸 IC。

### 4.3 成长因子

典型代表：

- 营收增速
- 扣非净利润增速
- ROE 改善
- 盈利预期上修

常见角色：

- 核心打分
- 时机叠加

成长因子的问题在于：

- 高成长很容易和高估值绑在一起
- 单独使用时，容易买到“好公司但太贵”

所以成长因子往往更适合：

- 与估值一起看
- 或在某个估值区间内再比较成长

### 4.4 动量因子

典型代表：

- 过去 1 个月收益
- 过去 3 个月收益
- 行业相对强度
- 盈利预期上修后的价格趋势

常见角色：

- 核心打分
- 时机叠加
- 最终排序加分项

动量因子和估值、质量很不一样，它更偏“市场行为”而不是“公司静态属性”。

动量因子通常更需要关注：

- 信号衰减速度
- 换手率
- 交易成本后是否仍有效
- 拥挤交易风险

所以动量因子通常应当在较宽的可交易股票池里先评测，再看放进策略后是否真的提升收益风险比。

### 4.5 波动率因子

典型代表：

- 过去 20 日波动率
- 下行波动率
- 特质波动率

常见角色：

- 风险惩罚
- 有时也可作为低波选股因子

这里要特别注意：

- 波动率因子不一定主要用来“预测收益”
- 很多时候它更重要的作用是“改善组合形态”

例如：

- 降低回撤
- 降低组合波动
- 提高夏普

所以波动率因子的评测方法，往往不能只看 RankIC，
还要看它加入组合以后：

- 夏普有没有提升
- 最大回撤有没有改善
- 换手是不是变差太多

## 5. 因子评测不是只分“全市场”还是“筛后市场”，而是两者都要看

这是最关键的一节。

### 5.1 结论先说

可以这样理解：

- 大多数因子，先在 `基础可交易池` 看原始能力
- 再在 `策略候选池` 看可用能力
- 最后在 `完整策略回测` 里看落地能力

所以不是二选一，而是三层递进。

### 5.2 哪些因子更应该先看宽股票池

通常更适合先在宽股票池里评测的因子有：

- 估值因子
- 动量因子
- 低波因子
- 流动性因子

原因是它们本来就更接近：

- 横截面排序信号
- 市场行为信号
- 组合风险信号

这类因子如果连在宽股票池里都完全没有表现，通常很难在后面突然变好。

但要注意，`宽股票池` 不是“毫无清洗的全体股票”，
而是至少要做基础可交易样本清洗。

### 5.3 哪些因子更应该做条件化评测

更适合做条件化评测的因子有：

- 质量因子
- 成长因子
- 红利可持续性因子
- 财务稳健性因子

原因是它们经常是“在某类股票内部更有效”，而不是“全市场裸用最有效”。

例如：

- 质量因子常常要在“估值不过分贵”的股票里才更有解释力
- 红利因子常常要在“已连续分红”的股票里才有意义
- 成长因子常常要在“非极端高估值”的样本里才更稳

### 5.4 一个实用的评测顺序

如果你要做得更像研究流程，可以固定成下面顺序：

1. `原始可交易池`：看原始 IC 和分组收益
2. `策略候选池`：看条件化后的 IC 和分组收益
3. `完整策略`：看回测收益、回撤、换手

这三个结果一起看，才比较接近真实。

## 6. 不同因子类别，是否应该有不同评测方法

答案是：应该有不同侧重，但不是完全不同学科。

所有因子共享一套基础工具：

- RankIC
- ICIR
- 正 IC 占比
- 分组收益
- 多空收益
- 回测验证

但不同类别的重点不同。

| 因子类别 | 更重要的评测重点 | 常见 horizon | 额外必须看 |
| --------- | ---------------- | ------------- | ------------ |
| 估值 | 中长周期 IC、分组收益、再定价能力 | 20/60/120/250 | 行业中性、价值陷阱 |
| 质量 | 条件化 IC、稳定性、组合回撤改善 | 60/120/250 | 披露时点、是否已被高估值覆盖 |
| 成长 | 增速持续性、预期修正、分组收益 | 20/60/120 | 估值约束、业绩兑现风险 |
| 动量 | 短中期 IC、衰减速度、交易成本后收益 | 5/20/60 | 换手、拥挤、反转风险 |
| 波动率 | 组合风险改善、下行保护、风险调整收益 | 20/60 | 是否只是风险因子而非 alpha 因子 |

所以更准确的说法是：

- 基础评测框架可以统一
- 但每一类因子的重点指标不一样

## 7. 一个更适合入门者的抽象框架

如果把整套流程抽象成一个知识框架，可以写成下面这条链：

### 7.1 第一步：先定义策略目标

你先要回答：

- 我做的是高股息价值策略，还是质量成长策略，还是动量策略？
- 我是月度调仓，季度调仓，还是半年调仓？
- 我更在意总收益，还是更在意低回撤？

如果这个问题没定，因子的研究标准会一直飘。

### 7.2 第二步：给每个因子先贴角色标签

例如：

- `dividend_yield`：核心打分
- `ep`：核心打分
- `roe_ttm`：质量过滤或辅助打分
- `momentum_20d`：时机叠加
- `volatility_20d`：风险惩罚

角色比名字更重要。

### 7.3 第三步：给每个因子指定评测股票池

例如：

- `ep`：宽股票池 + 策略候选池都测
- `roe_ttm`：宽股票池测原始能力，策略候选池测条件化能力
- `momentum_20d`：宽股票池重点测，策略内再测落地效果
- `volatility_20d`：重点看加入组合后是否改善回撤和夏普

### 7.4 第四步：单因子过关后，再谈多因子配权

多因子配权不是替代单因子评测，而是单因子评测之后的下一步。

常见顺序是：

1. 先看单因子是否有用
2. 再看因子之间是否高度重复
3. 再生成少量候选配权方案
4. 最后用完整回测比较哪套方案更好

### 7.5 第五步：不是所有因子都要进配权器

这是很多人刚入门时最容易犯的错误。

并不是所有因子都应该参与“权重优化”。

例如：

- `ST`、上市天数、分红连续性这类更适合做门槛
- `波动率` 更可能是惩罚项或上限约束
- 真正适合放进配权器的，通常是连续型核心打分因子

## 8. 当前仓库里的 `run_factor_test.py` 属于哪一层

当前脚本 [scripts/run_factor_test.py](../scripts/run_factor_test.py) 更接近：

- 针对一组 `核心打分因子`
- 做统一口径的单因子评测
- 再做少量规则化候选权重比较

它解决的是：

- 这几个打分因子谁更强
- 在指定 horizon 下，IC/ICIR 生成的权重方案是否比默认权重更好

它没有完整覆盖的是：

- 因子角色分层
- 宽股票池与条件股票池双重评测
- 动量因子的换手和交易成本检验
- 波动率因子的风险贡献检验
- 行业中性、规模中性、条件化回归等更细研究

所以这个脚本适合做第一版统一评测，不适合直接代表完整的因子研究体系。

## 9. 对你这次问题的直接回答

### 9.1 “是不是应该先用估值因子筛，再用质量成长，再用动量波动？”

可以作为某一类策略的实现方式，但不要把它当成普适顺序。

更稳的表达是：

- 先基础清洗
- 再硬过滤
- 再核心打分
- 再时机叠加和风险控制

在高股息价值策略里，估值确实常常更靠前；
但在趋势策略里，动量可能才是第一核心因子。

### 9.2 “因子的评测是不是应该在全股票池中？”

更准确的答案是：

- 应该先在基础可交易宽股票池中评测原始能力
- 再在策略候选池中评测可用能力

不是只看一个股票池，更不是裸用全市场全部股票。

### 9.3 “不同类别的因子是不是有不同评测方法？”

有不同重点，但底层框架可以统一。

统一的是：

- IC
- ICIR
- 分组收益
- 回测

不同的是：

- 估值更看长周期再定价
- 质量更看条件化有效性和回撤改善
- 动量更看衰减、换手和成本
- 波动率更看组合风险改善，而不只是 raw alpha

## 10. 给入门者的实操模板

如果你接下来想把研究流程固定下来，可以直接照着这张清单做。

### 10.1 先问自己 4 个问题

1. 我的策略目标是什么？价值、红利、成长、趋势还是混合？
2. 我的调仓频率是什么？
3. 每个因子在策略里是过滤、打分、时机还是风险控制？
4. 这个因子应该先在宽池评测，还是必须在条件池评测？

### 10.2 再给每个因子写一张研究卡片

每个因子至少写明：

- 因子类别
- 因子角色
- 评测股票池
- 评测 horizon
- 主要指标
- 是否纳入最终策略

### 10.3 最后把研究流程固定成 5 步

1. 设定股票池
2. 单因子评测
3. 条件化评测
4. 多因子候选配权
5. 完整策略回测

## 11. 一句话总结

你现在最需要建立的不是“哪类因子先、哪类因子后”的固定顺序，
而是这样一套判断框架：

- 先分股票池
- 再分因子角色
- 再选对应评测方法
- 最后才做多因子配权和策略回测

这样以后你无论研究估值、质量、成长、动量还是波动率，都会有一套统一骨架，不会每次都从头混乱开始。
