# 因子研究方法论

## 一句话定义

AI 在因子研究里负责提出假设、组织证据、生成候选、解释失败和设计下一轮实验；脚本、门禁和回测负责判断候选是否可靠。不能让智能体用语言直接替代实证结果。

## 核心原则

1. **研究从问题出发，不从公式出发**：先定义当前策略缺口，再寻找可能解释变量。
2. **候选必须可证伪**：每个因子都要有预测对象、时间边界、字段来源和失败条件。
3. **AI 只能提出候选，不能宣布有效**：有效性来自样本外、多周期、正交性、容量和策略增量贡献。
4. **坏因子也要归档**：失败原因是下一轮变异和避免重复试错的输入。
5. **因子不等于策略**：因子必须先判断策略角色，再进入信号或策略候选。
6. **研究自动化可以主动，实盘晋升必须保守**：研究可以批量跑，实盘必须门禁和人工审批。

## 研究对象

| 对象 | 含义 | 关键字段 |
|---|---|---|
| 研究议题 | 一个值得研究的问题或策略缺口 | 目标、约束、停止条件、优先级 |
| 研究假设 | 对市场行为的可证伪解释 | 经济逻辑、适用样本、预测周期 |
| 候选因子 | 可以计算的表达式、规则或代码片段 | 字段、表达式、方向、父代、风险 |
| 因子家族 | 同一逻辑下的一组候选 | 家族名、父代、变异路径、失败原因 |
| 评测运行 | 对候选做实证检查的一次运行 | 样本、参数、指标、产物 |
| 策略角色 | 因子在策略里的用途 | 排序、过滤、门控、风控、解释 |
| 版本血统 | 候选从何而来、怎样演化 | 输入、代码、数据、审批、产物 |

## Alpha 研究员工作协议

Alpha 研究员不是“找公式的人”，而是把市场现象、失败结果、数据字段和策略缺口变成可证伪候选的人。Agent 可以承担资料检索、候选草拟、公式变异、报告解释和反方审稿，但每个候选都必须先进入假设卡，再进入计算。

研究入口分四类：

| 入口 | 典型问题 | 产物 |
|---|---|---|
| 失败入口 | 强 IC 因子为什么 overlay 失败？收益牺牲来自过滤、状态误用还是风格重复？ | 失败复用假设 |
| 现象入口 | 市场里发生了什么可解释行为？追涨拥挤、公告漂移、流动性衰减是否可观察？ | 现象假设 |
| 数据入口 | 现有字段能观察什么行为？OHLCV、公告、资金流、分钟结构分别能表达什么？ | 字段到行为映射 |
| 组合入口 | 当前策略缺什么？回撤、左尾、换手、容量、可成交性还是候选池质量？ | 策略缺口假设 |

每个入口都必须回答：

1. 研究问题从哪里来。
2. 预期改善哪个策略问题。
3. 候选预期扮演什么角色。
4. 如果失败，失败经验如何进入下一轮。

## 假设卡

候选生成前必须先写假设卡。假设卡是 Alpha 研究员、Agent、脚本和 UI 的共同输入，不能用自然语言聊天记录替代。

```json
{
  "hypothesis_id": "hyp_20260516_001",
  "entry_type": "failure_reuse",
  "source_evidence": [
    "vtx_alpha_101 强 IC 但业绩预告漂移 overlay 收益牺牲过大"
  ],
  "market_behavior": "低风险/拥挤反转因子可能更适合识别事件策略左尾风险，而不是直接过滤候选池",
  "research_question": "只在高拥挤或弱市场状态下使用该因子，是否能降低坏持仓而不牺牲主 alpha？",
  "target_strategy": "earnings_forecast_drift_v3",
  "expected_role": "risk_budget",
  "required_fields": ["open", "high", "low", "close", "amount", "forecast_surprise"],
  "horizons": [5, 20],
  "candidate_budget": 6,
  "failure_conditions": [
    "降低收益但不能显著改善左尾",
    "只复现全市场 IC，不能改善目标策略",
    "效果主要来自小市值或低流动性暴露"
  ],
  "next_if_failed": "降级为解释变量或归档为 overlay_failed，不继续堆叠相似低风险公式"
}
```

## 生产线阶段

| 阶段 | 目标 | 主责智能体 | 产物 |
|---|---|---|---|
| 问题定义 | 明确为什么要研究 | 研究总监 | 研究议题卡 |
| 证据检索 | 查已有档案、论文、失败路径 | 归档审计员 | 证据包 |
| 数据契约 | 判断字段是否存在、可见、覆盖足够 | 数据运营负责人 | 数据可用性报告 |
| 假设会议 | 形成可证伪假设和停止条件 | 研究总监 / CogAlpha 负责人 | 会议结论 |

| 静态门禁 | 检查字段、PIT、表达式、重复度 | 因子质量审查员 | 门禁报告 |
| 沙箱计算 | 在隔离环境计算因子值 | 数据运营负责人 | 因子值快照 |
| 预测评测 | 多周期 IC、多空、分层、换手 | 研究总监 | 评测报告 |
| 证据评审 | 判断研究结论是否可信、是否应接力 | 因子研究证据评审员 | 证据评审和接力包 |
| 稳健性评估 | 样本外、分市场、分行业、分市值 | 风险官 | 稳健性报告 |
| 正交性评估 | 判断与现有因子和策略的增量贡献 | 策略晋升专员 | 增量价值报告 |
| 策略角色判定 | 决定用途和是否晋升 | 策略晋升专员 / 风险官 | 策略角色建议 |
| 归档与演化 | 记录成功、失败、下一轮变异 | 归档审计员 / CogAlpha 负责人 | 因子档案和演化队列 |

## 研究议题卡

每个研究议题至少包含：

```json
{
  "议题编号": "topic_20260516_001",
  "标题": "量价低波拥挤反转是否能改善当前策略容错率",
  "来源": "当前策略在高拥挤阶段回撤变大",
  "目标": "寻找能过滤拥挤或改善排序的候选因子",
  "预测周期": [5, 20],
  "适用股票池": "默认可交易 A 股股票池",
  "停止条件": [
    "连续 20 个候选无样本外增量",
    "字段覆盖无法满足评测",
    "与现有因子相关性过高"
  ],
  "风险假设": ["小市值暴露", "容量不足", "涨跌停不可成交"]
}
```

## 候选因子契约

候选因子不是一句自然语言想法，必须是结构化对象：

```json
{
  "候选编号": "alpha_20260516_001",
  "因子家族": "liquidity_impact_reversal",
  "研究假设": "短期流动性冲击后，价格反转可能提供排序信息",
  "表达式类型": "公式",
  "表达式": "rank(-rolling_corr(turnover, return, 20))",
  "所需字段": ["成交额", "成交量", "收盘价"],
  "字段可见时间": "交易日收盘后",
  "预测周期": [5, 20],
  "预期角色": "过滤条件",
  "预期方向": "数值越高，未来收益越高",
  "父代候选": ["alpha_liquidity_impact_v1"],
  "生成方式": "CogAlpha 变异",
  "失败条件": ["只在单一年份有效", "接入目标策略后收益牺牲过大"],
  "禁止事项": ["使用未来收益", "使用未来价格", "跨期回填"],
  "主要风险": ["行业暴露", "小市值暴露", "容量不足"]
}
```

## 质量门禁

质量门禁只回答“能不能评测”，不回答“有没有收益”。

硬阻断：

1. 字段在预测时点不可见。
2. 使用未来收益、未来价格、未来公告或跨期回填。
3. 表达式不可复现，或依赖任意网络/随机外部状态。
4. 覆盖率过低，缺失值或无穷值不可控。
5. 与已有因子近似重复，且没有新的经济解释。
6. 代码候选未进入沙箱。

软警告：

1. 行业、市值、流动性暴露过强。
2. 换手可能过高。
3. 预测周期与经济逻辑不匹配。
4. 解释强但样本覆盖不足。
5. 需要额外成交约束才能交易。

## 实证评测指标

| 指标组 | 指标 | 用途 |
|---|---|---|
| 预测力 | IC、RankIC、ICIR、IC 胜率 | 判断排序能力 |
| 收益形态 | 多空收益、分层收益、单调性 | 判断是否有可交易结构 |
| 稳定性 | 分年、分市场状态、分行业、分市值 | 判断是否依赖单一环境 |
| 成本敏感 | 换手、冲击成本、成交额参与率 | 判断能否交易 |
| 风险暴露 | 行业、市值、波动、流动性、估值 | 判断是否只是风格暴露 |
| 正交性 | 与已有因子相关性、残差 IC | 判断是否有新增信息 |
| 策略贡献 | 加入现有策略后的收益、回撤、换手变化 | 判断是否值得晋升 |

最低要求不是“某个指标漂亮”，而是指标之间不能互相矛盾。

## 多周期评测

同一候选至少检查：

1. 1 日、5 日、10 日、20 日、60 日预测周期。
2. 全样本和样本外。
3. 牛市、熊市、震荡、极端流动性阶段。
4. 主要指数成分、全市场、默认可交易股票池。
5. 行业中性和非行业中性两种口径。
6. 交易约束前和交易约束后。

如果只在单一周期有效，默认不能晋升，只能进入观察或变异。

## 正交性和增量贡献

一个因子可能“单独有效”，但对当前策略没有价值。必须检查：

1. 与已有因子的相关性。
2. 对已有信号的残差解释力。
3. 加入策略后的收益、回撤、换手、容量变化。
4. 是否只是在重复行业、市值或流动性暴露。
5. 是否能改善当前策略的具体问题，例如回撤、空仓、拥挤、不可成交。

只有能解释“为什么对现有系统有新增价值”的候选，才进入策略候选。

## Agent 接力工作法

Agent 的价值不是替代研究员判断，而是把研究动作拆成可审稿、可追溯、可继续的接力。每次接力都围绕同一张假设卡，不允许因为一个实验失败就随机换题。

### 接力顺序

```text
研究总监
  -> 归档审计员
  -> CogAlpha 负责人
  -> 因子质量审查员
  -> 多周期评测
  -> 因子研究证据评审员
  -> CogAlpha 负责人 / 策略晋升专员 / 风险官 / 归档审计员
```

关键点：

1. CogAlpha 负责人负责“提出下一批候选”，不负责宣布有效。
2. 因子质量审查员负责“能不能评测”，不负责解释收益。
3. 因子研究证据评审员负责“信不信、要不要继续、交给谁”。
4. 策略晋升专员只在证据评审后接手，判断策略角色。
5. 归档审计员不仅归档成功，也关闭无效路径。

### 时间不是调度器

30 分钟、1 小时、2 小时不是三套产品版本，而是一个研究员能连续推进的深度。

| 时间深度 | Alpha 研究员应该完成什么 |
|---|---|
| 30 分钟 | 读旧档案、复核一个结果、写出下一位 Agent 接力包 |
| 1 小时 | 完成一张假设卡、一组候选或一次证据评审 |
| 2 小时以上 | 围绕同一假设完成多轮变形、反方审稿和归档 |

如果会话很长，研究员每轮都要写“本轮学到什么、下一轮为什么值得继续”。如果写不出来，就触发停止条件。

## 策略角色判定

| 角色 | 晋升方向 | 判断重点 |
|---|---|---|
| 排序因子 | 信号候选 | 多周期排序能力和稳定性 |
| 过滤条件 | 策略规则候选 | 是否减少坏交易、不牺牲过多机会 |
| 状态门控 | 市场状态模块 | 是否降低回撤和错误开仓 |
| 风险预算 | 仓位或集中度模块 | 是否改善回撤、容量和拥挤 |
| 执行门禁 | 交易执行模块 | 是否减少不可成交、滑点和废单 |
| 解释变量 | 归因模块 | 是否帮助理解策略而不直接交易 |

同一个因子可以从“排序因子”降级为“解释变量”或“过滤条件”。这不是失败，是正确归位。

## 智能体分工边界

| 智能体 | 能做什么 | 不能做什么 |
|---|---|---|
| 研究总监 | 定义议题、停止条件、优先级 | 直接宣布因子有效 |
| CogAlpha 负责人 | 生成候选、变异、交叉、父代血统 | 跳过质量门禁 |
| 因子质量审查员 | 阻断不可评测候选 | 因为故事好听放行 |
| 因子研究证据评审员 | 审稿评测结论、识别过拟合和下一位 Agent | 生成新因子或直接晋升策略 |
| 数据运营负责人 | 检查字段、快照、计算可复现 | 临时补脏数据继续跑 |
| 风险官 | 审查未来函数、过拟合、容量、实盘风险 | 为了收益放松硬门禁 |
| 策略晋升专员 | 判断策略角色和晋升路径 | 让因子直接进入实盘 |
| 归档审计员 | 记录结论、失败、版本血统 | 只归档成功结果 |

## UI 推导

因子研究实验室至少需要这些页面：

1. 研究议题队列：为什么研究、目标是什么、什么时候停止。
2. 研究入口看板：失败、现象、数据、组合缺口四类入口及其证据。
3. 假设卡详情：市场行为、目标策略、预期角色、字段、失败条件、候选预算。
4. 候选生成会议：哪些智能体参与，分歧是什么，生成了哪些候选。
5. 候选因子详情：表达式、字段、方向、父代、风险、当前状态。
6. 质量门禁面板：硬阻断、软警告、证据和补证据任务。
7. 评测结果面板：多周期、分层、暴露、成本、稳定性。
8. 正交性和策略贡献面板：是否对当前策略有新增价值。
9. 策略角色判定页：排序、过滤、门控、风控、执行或解释。
10. 因子档案页：成功、失败、变异、下次研究队列。

当前已有总览原型只能验证方向，不能验证完整研究工作。下一批原型应该先补“候选因子详情”和“质量门禁面板”。
