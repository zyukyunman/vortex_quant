# 通知与手机触达

## 一句话定义

通知机制不是另一个复杂 App。第一版把独立飞书应用当作个人移动控制入口：系统只在任务阻塞、需要审批、交易异常、研究有重要结论或事故复盘时主动触达你，并允许你在飞书内完成低风险交互。

## 设计原则

1. **默认静默运行**：没有风险、没有审批、没有异常时不打扰。
2. **飞书应用是移动入口，不是完整工作台**：飞书负责提醒、确认、追问和低风险决策；复杂审查仍回到工作台。
3. **每条通知能回到事实源**：必须关联运行、产物、决策或智能体会议。
4. **实盘相关高优先级**：交易阻断、账户异常、下单失败、对账不一致必须推送。
5. **个人使用优先简单**：默认使用你新建的独立飞书应用，不和工作飞书应用混用。
6. **每个交互都要写回控制面**：飞书按钮、指令和回复必须转成决策、任务或审计记录。

## 触发事件

| 事件 | 级别 | 是否通知手机 | 默认动作 |
|---|---|---:|---|
| 实盘/模拟盘执行失败 | 紧急 | 是 | 立即通知，进入交易审查 |
| QMT 桥接掉线或认证失败 | 紧急 | 是 | 阻断交易，等待处理 |
| 订单/成交/持仓对账不一致 | 紧急 | 是 | 阻断后续下单 |
| 数据更新失败且影响盘前策略 | 高 | 是 | 通知并生成修复任务 |
| 研究发现可晋升候选 | 中 | 是 | 通知摘要，等待你决定是否看 |
| 因子评测普通完成 | 低 | 可选 | 写入工作台，不主动打扰 |
| 智能体会议有重大分歧 | 中 | 是 | 请求你追问或指定取舍 |
| 周期性复盘生成 | 低 | 可选 | 工作台展示 |

## 通知内容格式

```text
标题：交易审查阻断：QMT 桥接认证失败
级别：紧急
影响：今日业绩预告策略无法继续自动提交委托
证据：运行 run_...，错误 ...
建议：暂停交易链路，检查 QMT token 和 bridge 状态
飞书动作：确认已知悉 / 查看运行 / 请求智能体诊断 / 暂停链路
```

## 飞书动作

第一版飞书只允许低风险动作：

1. 查看摘要。
2. 确认已知悉。
3. 驳回或暂停研究任务。
4. 追问智能体。
5. 批准低风险离线研究继续运行。
6. 请求生成诊断报告。
7. 打开本地工作台链接或查看运行编号。

飞书默认不允许：

1. 真实账户下单。
2. 修改实盘配置。
3. 修改 API key 或账户。
4. 删除受保护产物。
5. 将策略推进实盘。
6. 关闭交易风控阻断。

## 飞书指令和卡片动作

| 命令 | 用途 | 是否允许改变系统状态 |
|---|---|---:|
| `/status` | 查看当前运营状态简报 | 否 |
| `/runs` | 查看阻塞、运行中、等待审批的运行 | 否 |
| `/approvals` | 查看待处理事项 | 否 |
| `/run run_id` | 查看某个运行摘要和证据链接 | 否 |
| `/ack event_id` | 确认已知悉某条通知 | 是，低风险 |
| `/ask run_id 问题` | 对某个运行追问智能体 | 是，生成追问任务 |
| `/pause run_id` | 暂停研究或离线任务 | 是，限非交易链路 |
| `/help` | 查看可用命令 | 否 |

高风险动作必须把飞书回复转成“待审批请求”，再由工作台确认。

## 渠道策略

第一版：

1. 独立飞书自建应用作为主通道。
2. 通知中附带运行编号、产物编号、工作台链接和低风险按钮。
3. 高风险事件重复提醒，直到确认或状态解除。
4. Mac Mini 优先使用飞书长连接或中间转发接收交互事件，避免第一版就暴露公网回调地址。

第二版：

1. 增加可插拔备用通道，例如 Bark、Pushover、iMessage 或邮件。
2. 每个渠道实现统一 `通知渠道` 协议。
3. 支持静默时段、重复提醒、升级通知。
4. 支持更细的卡片交互，但不把实盘危险动作放进手机端。

本文冻结第一版默认选择：**独立飞书应用是主通道，其他渠道只做备用**。

## 飞书约束

1. 需要使用独立飞书自建应用，启用机器人能力或消息发送能力。
2. 只允许白名单飞书用户或会话交互。
3. App ID、App Secret 和访问令牌只能放在本机密钥文件或环境变量，不能写入仓库。
4. 每条飞书事件或卡片回调必须做幂等处理，避免重复点击造成重复决策。
5. 交易链路通知必须支持二次确认和审计记录。

## 通知和工作台的关系

飞书负责“叫你回来”和处理轻量动作，工作台负责“解释发生了什么”。飞书不能承载完整上下文，也不能替代运行中心。

```text
事件
  -> 通知路由
  -> 飞书消息 / 卡片
  -> 用户确认、追问、暂停或打开工作台
  -> 控制面写入决策/任务/审计
  -> 工作台展示运行/产物/智能体会议/决策
```
