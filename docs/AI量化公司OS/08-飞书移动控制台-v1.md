# 飞书移动控制台

## 一句话定义

飞书移动控制台是 Vortex AI 量化公司的个人远程入口。它基于你新建的独立飞书应用，不与工作飞书应用混用；负责通知、轻量查询、低风险决策和追问智能体，不直接执行高风险交易动作。

## 产品定位

飞书应用不是“另一个完整前端”，而是移动端操作层：

```text
系统事件
  -> 通知路由
  -> 飞书消息 / 交互卡片 / 指令
  -> 用户轻量动作
  -> 控制面决策或任务
  -> 工作台和归档中心展示事实源
```

它解决三个问题：

1. 你不在 Mac 前时，系统可以主动叫你回来。
2. 你可以在手机飞书里快速判断“要不要处理”。
3. 你可以把问题丢回智能体，而不是立即打开电脑查日志。

## 第一版能力边界

| 能力 | 第一版是否支持 | 说明 |
|---|---:|---|
| 关键事件通知 | 是 | 阻塞、交易异常、待审批、重要研究结论 |
| 查询当前状态 | 是 | 通过指令或卡片按钮返回运营状态简报 |
| 查询运行 | 是 | 返回运行摘要和证据链接 |
| 确认已知悉 | 是 | 写入通知确认记录 |
| 追问智能体 | 是 | 生成追问任务，进入智能体会议或诊断流程 |
| 暂停离线研究任务 | 是 | 只允许非交易链路 |
| 批准低风险离线研究 | 是 | 例如继续跑批量评测 |
| 修改实盘配置 | 否 | 必须回工作台 |
| 下单/撤单 | 否 | 禁止 |
| 策略晋升到实盘 | 否 | 必须回工作台审批 |

## 指令和卡片动作

| 入口 | 返回内容 | 后续动作 |
|---|---|---|
| 状态简报 | 实盘、数据、研究、策略、智能体、通知状态 | 打开工作台 / 查看阻塞 |
| 运行列表 | 阻塞、警告、待审批运行列表 | 查看某个运行 |
| 运行详情 | 当前阶段、失败点、产物链接、建议动作 | 追问 / 确认 / 打开工作台 |
| 待审批 | 当前待处理事项 | 批准低风险项 / 驳回 / 追问 |
| 追问智能体 | 已创建追问任务 | 等待智能体回复 |
| 确认通知 | 通知已确认 | 写入决策记录 |
| 暂停运行 | 尝试暂停离线任务 | 成功/失败结果回推 |

飞书第一版优先使用“消息卡片按钮 + 简单文本指令”，不急着做复杂表单。

## 消息模板

### 交易异常

```text
级别：紧急
事件：QMT 桥接认证失败
影响：今日模拟盘链路不会继续提交新委托
证据：run_20260516_trade_qmt / task_bridge_health
建议：暂停交易链路，检查 QMT 认证状态

按钮：确认已知悉 / 请求诊断 / 打开工作台
```

### 因子研究结论

```text
级别：中
事件：因子候选完成多周期评测
结论：流动性冲击反转通过基础门禁，但容量风险偏高
证据：factor_alpha_liquidity_impact_v3 / eval_run_...
建议：进入正交性和组合贡献评估

按钮：查看摘要 / 追问风险 / 驳回候选 / 进入下一步
```

### 运行阻塞

```text
级别：高
事件：研究运行等待字段覆盖证据
阻塞：路径质量门控候选缺少分钟成交结构覆盖率
建议：让数据运营负责人补证据，或驳回候选

按钮：追问智能体 / 暂停运行 / 驳回候选
```

## 交互状态机

```text
待发送
  -> 已发送
  -> 用户已查看
  -> 用户已确认 / 已追问 / 已暂停 / 已驳回
  -> 控制面已写入
  -> 工作台已同步
```

飞书消息本身不能作为唯一事实源。真正状态以控制面为准。

## 安全边界

1. 只允许你的飞书应用和白名单用户交互。
2. 每个按钮都携带一次性 action id。
3. action id 必须绑定事件、运行、用户和过期时间。
4. 重复点击同一个按钮只返回已有结果，不重复执行。
5. 高风险动作只创建审批请求，不直接执行。
6. App ID、App Secret、tenant_access_token 不进入 Git，不进入日志，不进入研究产物。
7. 飞书消息不包含完整密钥、账户号、持仓敏感明细。

## 技术建议

第一版建议用“自建应用 + 应用身份发消息 + 事件回调或长连接接收用户交互”：

```text
Mac Mini 后台进程
  -> 获取 tenant_access_token
  -> 调用飞书消息接口发送文本或交互卡片
  -> 接收飞书事件或卡片回调
  -> 校验白名单和 action id
  -> 写入 control.db
  -> 调用本地任务/决策接口
  -> 回发飞书结果
```

如果本机没有公网入口，第一版优先考虑飞书长连接或中间转发服务；不要为了回调过早暴露 Mac Mini。

## 与工作台的关系

飞书只回答“我现在要不要处理”。工作台回答“为什么发生、证据在哪里、下一步怎么治理”。

UI 原型需要补两类页面：

1. 飞书消息详情和按钮状态。
2. 工作台里的通知审计页，展示每条飞书消息、送达状态、用户动作和控制面决策。
