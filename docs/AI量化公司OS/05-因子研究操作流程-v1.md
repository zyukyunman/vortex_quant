# 因子研究操作流程

## 一句话定义

因子研究不是“让模型随机想因子”，而是一条可运营的生产线：提出研究议题、检索已有证据、生成候选、审查质量、跑实证、判断策略角色、归档经验，并把成功和失败都反馈给下一轮。

产品和 UI 必须先理解这条流程，再设计研究实验室和智能体办公室。

## 流程总览

```text
研究议题池
  -> 资料和档案检索
  -> 数据可用性检查
  -> 研究委员会
  -> 候选生成
  -> 质量门禁
  -> 多周期评测
  -> 证据评审
  -> 正交性和组合贡献
  -> 策略角色判定
  -> 晋升 / 变异 / 拒绝 / 归档
```

## 阶段定义

| 阶段 | 主责岗位 | 输入 | 输出 | UI 必须展示 |
|---|---|---|---|---|
| 研究议题池 | 研究总监 | 当前策略瓶颈、历史失败、外部资料 | 研究议题 | 优先级、目标、停止条件 |
| 资料和档案检索 | 归档审计员 | 因子档案、策略结论、论文/资料 | 已知证据 | 已做过什么、失败原因 |
| 数据可用性检查 | 数据运营负责人 | 字段、快照、可见时间 | 数据可用性结论 | 缺字段、PIT 风险、覆盖率 |
| 研究委员会 | 研究总监 / CogAlpha PI | 议题和证据 | 候选生成计划 | 参会角色、分歧、决策 |
| 候选生成 | CogAlpha PI | 研究方向、父代、约束 | 因子候选 | 假设、字段、周期、来源视角 |
| 质量门禁 | 因子质量审查员 / 风险官 | 候选表达和字段 | 通过/拒绝/补证据 | 阻断项、警告项、证据 |
| 多周期评测 | 研究总监 | 通过门禁的候选 | IC、多空、稳定性 | 指标、样本边界、分组表现 |
| 证据评审 | 因子研究证据评审员 | 评测结果、失败原因、父代血统 | 可信/薄弱证据和下一位 Agent | 证据强弱、过拟合风险、接力对象 |
| 正交性和组合贡献 | 策略晋升专员 | 候选和现有因子池 | 增量价值判断 | 相关性、重复度、组合贡献 |
| 策略角色判定 | 策略晋升专员 / 风险官 | 评测结果 | 排序/过滤/门控/风控腿 | 建议角色、适用场景 |
| 归档和演化 | 归档审计员 / CogAlpha PI | 成功和失败结论 | 版本链和下一轮队列 | 父代、子代、失败原因 |

## 智能体分工

| 岗位 | 主要问题 | 是否有阻断权 |
|---|---|---:|
| 研究总监 | 这个方向是否值得投入？ | 是 |
| CogAlpha PI | 应该调用哪些研究视角，如何变异和交叉？ | 否 |
| 因子质量审查员 | 候选是否可计算、可见、稳定、无明显作弊？ | 是 |
| 因子研究证据评审员 | 评测结论是否可信，下一步交给谁？ | 是 |
| 风险官 | 是否有未来函数、过拟合、容量或实盘风险？ | 是 |
| 策略晋升专员 | 因子应该扮演什么策略角色，是否进入下一阶段？ | 是 |
| 归档审计员 | 结论是否可追溯，失败经验是否沉淀？ | 是 |

## 候选因子的最小结构

```json
{
  "候选编号": "alpha_20260516_001",
  "研究假设": "低流动性冲击后的短期反转可能提供排序信息",
  "来源视角": "流动性 / 反转 / 拥挤",
  "所需字段": ["收盘价", "成交额", "成交量"],
  "预测周期": [5, 20],
  "表达形式": "公式或受限代码",
  "预期方向": "数值越高预期收益越低",
  "父代候选": ["alpha_liquidity_impact_v1"],
  "主要风险": ["小市值暴露", "容量不足", "涨跌停不可成交"]
}
```

## 质量门禁

候选进入评测前必须回答：

1. 字段在交易时点是否可见。
2. 是否使用未来收益、未来价格或负向移动。
3. 覆盖率、缺失值、无穷值、唯一值比例是否达标。
4. 是否与已有因子高度重复。
5. 经济解释是否和表达式一致。
6. 如果是代码候选，是否只在隔离研究沙箱执行。

质量门禁只证明“可以评测”，不证明“有效”。

## 证据评审和接力

多周期评测完成后，不能直接进入策略晋升。必须先由因子研究证据评审员判断：

1. 结果是否回答原始假设，而不是换了问题。
2. 是否只在单一周期、单一年份、单一参数窗口有效。
3. IC、多空、分层、策略贡献是否相互支持。
4. 与已有因子或父代是否高度重复。
5. 失败是否有信息价值，能否指导变形。
6. 下一步应该交给谁。

接力规则：

| 证据评审结论 | 下一位 Agent | 说明 |
|---|---|---|
| `invalid_experiment` | 因子质量审查员 | 修 PIT、字段、表达式或覆盖率；不能继续解释收益 |
| `no_signal` | 归档审计员 | 关闭路径，记录为什么失败 |
| `informative_failure` | CogAlpha PI | 围绕失败原因做变形或方向翻转 |
| `research_lead` | CogAlpha PI / 研究总监 | 继续同一假设的子代，不换题 |
| `candidate_role_unclear` | 策略晋升专员 | 判断它是排序、过滤、门控、风控还是解释变量 |
| `risk_unclear` | 风险官 | 审查样本外、容量、回撤、成本和过拟合 |
| `execution_unclear` | 执行运营 | 只处理可交易性，不创造 alpha |

这个接力可以由 Agent 在 markdown 摘要中完成，不要求写成代码调度。产品上只需要把“当前结论、下一位 Agent、为什么、下一步产物”显示清楚。

## 策略角色判定

一个因子不能只按“好/坏”判断，还要判断用途：

| 角色 | 含义 | 例子 |
|---|---|---|
| 排序因子 | 直接参与股票排序 | 盈利 surprise 排名 |
| 过滤条件 | 剔除不适合交易的样本 | ST、低流动性、极端波动 |
| 状态门控 | 决定策略是否打开 | 市场趋势、波动状态 |
| 风险预算 | 调整仓位或集中度 | 容量、回撤、拥挤度 |
| 执行门禁 | 影响是否下单或如何下单 | 涨跌停、开盘流动性 |
| 解释变量 | 用于归因，不直接交易 | 行业/市值/估值暴露 |

## UI 推导

研究实验室的页面顺序应按流程走，而不是按代码模块走：

1. 研究方向队列。
2. 候选生成会议。
3. 质量门禁面板。
4. 多周期评测面板。
5. 正交性和组合贡献面板。
6. 证据评审和 Agent 接力面板。
7. 策略角色判定面板。
8. 归档和下一轮演化。

智能体办公室只展示“谁在做什么”和“争论结果”，不能替代研究流程本身。
