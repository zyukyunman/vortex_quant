# 运行与产物契约

## 一句话定义

运行、任务、产物、决策、智能体会议和版本血统是 AI 量化公司操作系统的事实源。界面、智能体、通知和自动化都只能围绕这些对象读写状态，不能把自由文本聊天当作系统状态。

## 对象模型

```text
运行
  -> 任务[]
  -> 产物[]
  -> 决策[]
  -> 智能体会议[]
  -> 版本血统
```

| 对象 | 含义 | 存储建议 |
|---|---|---|
| 运行 | 一次可追踪的业务运行 | control.db + 运行清单 |
| 任务 | 运行内的步骤 | control.db |
| 产物 | 运行产生的结构化产物 | 工作区文件 + 产物清单 |
| 决策 | 审批、驳回、晋升、暂停、复盘 | control.db + 审计日志 |
| 智能体会议 | 多智能体会议或诊断过程 | JSON 产物 + 摘要 |
| 版本血统 | 输入、输出、代码、参数、审批链 | 产物清单 |

## 状态枚举

### 运行状态

| 状态 | 含义 | 是否终态 |
|---|---|---:|
| 待调度 | 已登记，等待调度 | 否 |
| 运行中 | 正在执行 | 否 |
| 等待中 | 等待数据、市场、审批或外部服务 | 否 |
| 已阻断 | 被门禁、风险或缺失条件阻断 | 否 |
| 已完成 | 正常完成 | 是 |
| 已失败 | 执行失败 | 是 |
| 已取消 | 人工或系统取消 | 是 |
| 已过期 | 心跳过期，需要分诊 | 否 |

### 任务状态

`待调度 -> 运行中 -> 已完成 / 已失败 / 已阻断 / 已跳过 / 已取消`

### 决策类型

| 类型 | 含义 |
|---|---|
| 批准 | 批准继续 |
| 驳回 | 驳回 |
| 追问 | 追问或要求补证据 |
| 暂停 | 暂停 |
| 晋升 | 晋升到下一阶段 |
| 回滚 | 回滚到上一版本 |
| 知悉 | 已知悉，不触发动作 |

## 版本血统要求

每个可晋升产物必须记录：

1. 使用的数据快照。
2. 使用的 profile / preset / config。
3. 产生它的 run id。
4. 关键代码版本或 code hash。
5. 上游产物编号。
6. 审批决策编号。
7. 下游消费者。

禁止：

1. 使用 `latest` 作为不可追溯输入。
2. 只有 HTML 报告、没有 canonical JSON。
3. 智能体会议结论没有结构化摘要。
4. 策略或交易产物没有审批记录。

## 接口适配层

第一版本地接口只做控制面适配层：

| 接口 | 用途 |
|---|---|
| `GET /api/brief/current` | 运营驾驶舱当前状态简报 |
| `GET /api/runs` | 运行中心列表 |
| `GET /api/runs/{run_id}` | 运行详情 |
| `GET /api/artifacts` | 产物搜索 |
| `GET /api/agent-sessions` | 智能体会议列表 |
| `GET /api/approvals` | 待审批 inbox |
| `POST /api/decisions` | 写入用户决策 |
| `GET /api/events/stream` | SSE/WebSocket 状态推送 |

接口不能绕过领域模块直接改数据面；它只提交控制面动作或读取事实源。

## 与现有 Vortex 的衔接

现有 `state/control.db`、`task_queue`、工作区产物和 CogAlpha `generation_summary.json` 已经接近本契约的底层形态。后续实现应优先新增适配层和补齐契约，不推翻数据、研究、策略、交易的域边界。
