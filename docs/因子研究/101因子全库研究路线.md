# 101 因子全库研究路线

关联：[[动量与101量价因子]]、[[因子方法论与多周期组合]]、[[外部Alpha资料线索]]

## 一句话结论

101 因子目前还没有“全库研究完”。已经验证的是一批 101 风格量价组件和组合残差；下一步要把全库拆成可计算算子、公式队列、评测矩阵和 Obsidian 档案，而不是只研究几个看起来强的公式。

1. 初步研究了低波、短反、成交拥挤反转、量价相关、价值动量低波等 101 风格组件。
2. 构造过严格低相关组合，并做过市值/波动/换手/流动性/短反转正交。

8. 新增 Batch D 的 10 个 Vortex Alpha101-style 公式，并完成 2017-2026 日频复验，覆盖日 K 形态、影线、跳空、区间扩张和真实波幅低风险结构。

## 全库拆解方式

| 类别 | 典型算子 | 研究重点 |
|---|---|---|
| 横截面排序 | `rank`、`scale`、`signed_power` | 当日截面方向和风格暴露 |
| 时间序列状态 | `ts_rank`、`delta`、`delay`、`decay_linear` | 半衰期、持有期、衰减曲线 |
| 量价关系 | `correlation`、`covariance`、volume/amount | 成交拥挤、放量延续或反转 |
| 价格结构 | open/high/low/close、returns | 日内强弱、影线、跳空、反转 |
| 中性化 | 行业、市值、流动性、波动、换手 | 原始 alpha 与残差 alpha 分离 |

## 每个公式的归档字段

| 字段 | 含义 |
|---|---|
| `formula_id` | Alpha 编号或 Vortex 自定义编号 |
| `formula_family` | 动量、反转、量价相关、波动、形态、混合 |
| `required_fields` | open/high/low/close/volume/amount/returns/adv 等 |
| `pit_safe` | 是否只使用 T 日及以前数据 |
| `coverage` | 有效覆盖率和缺失原因 |
| `horizon_fit` | 适合 1/5/10/20/60 哪些期限 |
| `ic_profile` | IC、ICIR、正 IC 占比、年度分段 |
| `layer_profile` | 分层收益和多空收益 |
| `style_exposure` | 市值、流动性、波动、换手、行业暴露 |
| `residual_ic` | 风格中性化后是否还有效 |
| `turnover_cost` | 换手和成本敏感性 |
| `role` | 排序、过滤、风险、择时、组合腿、拒绝 |
| `strategy_hook` | 可接入的策略位置 |

## 研究顺序

1. **算子层**：先把 Alpha101 风格的公共算子变成测试覆盖的 Vortex 函数。
2. **公式队列**：按类别录入公式，不一次性把公式和研究结果混在脚本里。
3. **批量计算**：逐批计算，每批生成因子矩阵和 coverage。
4. **批量评估**：统一跑 IC、分层、多空、暴露、残差 IC、换手。
5. **角色分类**：按结果分成排序、过滤、风险、择时、组合腿和拒绝。
6. **组合实验**：只把通过角色分类的因子接入业绩预告漂移、质量红利腿或短反腿。
7. **归档**：好坏都写进因子档案，避免重复研究。

## 对策略的接入原则

1. 不把 101 单公式直接当策略。
2. 不把强 IC 公式直接塞进 live 默认排序。
3. 先看它在主策略里的边际贡献：
   - 是否减少坏持仓。
   - 是否降低弱市左尾。
   - 是否提高候选池质量。
   - 是否降低换手或改善成交。
4. 原始公式和中性化残差要同时评估；如果原始版有效、残差版无效，就标注为风格代理。
