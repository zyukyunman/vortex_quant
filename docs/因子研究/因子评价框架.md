# 因子评价框架

关联：[[因子研究档案]]、[[因子研究档案规范]]、[[因子实验总表]]

## 一句话结论

单因子不能用“裸跑 TopN 是否低回撤”一票否决。因子评价要先判断它的角色，再看它在组合中的边际贡献：排序、过滤、择时、仓位缩放、风险状态、容量和交易成本。

## 因子角色

| 角色 | 评价重点 | 不能误用的标准 |
|---|---|---|
| Alpha 排序因子 | RankIC、分层收益、横截面稳定性、行业/风格中性后残差 IC | 不要求单因子裸跑就满足最终策略回撤 |
| 过滤因子 | 过滤前后胜率、尾部亏损、换手、覆盖率 | 不要求过滤后的收益一定最高，关键看是否去掉坏尾部 |
| 择时/状态因子 | 对策略收益的 exposure scaling、弱市保护、状态转移稳定性 | 不应只看单因子选股 IC |
| 仓位因子 | 仓位缩放后的 Calmar、最大回撤、暴露均值、踏空成本 | 不应只看空仓后回撤下降 |
| 风险因子 | 与亏损日、波动、拥挤、流动性冲击的相关性 | 不要求它正向预测收益 |
| 组合腿 | 与已有因子的相关性、正交后 IC、组合边际收益/回撤改善 | 不要求独立成为策略 |

## 标准评测矩阵

1. **预测力**：1d/3d/5d/10d/20d RankIC、ICIR、正 IC 占比、分层收益。
2. **稳定性**：年度、月度、牛熊/震荡/下跌状态分段，避免只吃单一年份。
3. **正交性**：与市值、波动、换手、动量、价值、质量、行业暴露的相关性和残差 IC。
4. **组合贡献**：接入已有策略后，比较年化、回撤、Sharpe、Calmar、换手、暴露、坏日收益。
5. **交易可行性**：调仓延迟、滑点、成本、容量、成交量约束、开盘/收盘可成交性。
6. **稳健性**：不同 universe、不同参数、不同 rebalance、不同成本和样本外区间。
7. **失效诊断**：如果裸跑回撤大，要拆解是市场 beta、风格暴露、集中度、换手还是因子方向问题。

## 对峰岭谷因子的修正口径

峰岭谷成交量结构是分钟微观结构因子，不应只按“`ridge_minutes` TopN 多头最大回撤”判断。正确口径是：

1. `ridge/peak/valley` 对个股未来收益有没有横截面排序价值。
2. 全市场 `ridge/peak/valley` 聚合是否能识别风险状态。
3. 在市场动量下行时，峰谷状态能否替代“硬空仓”为半仓、换因子或防守持仓。
4. 接入业绩预告漂移、质量价值、低波反转后，是否提升组合 Calmar 或降低坏日亏损。
5. 如果降低回撤但牺牲收益，要比较它与大盘动量门控的边际差异，而不是简单判定有效/无效。

## 文献与研究框架参考

1. Grinold/Kahn 的主动管理框架强调信息系数、广度、约束和组合构建，适合把单因子视为组合输入，而不是直接等同策略。
2. Qian/Hua/Sorensen 的组合管理框架强调 alpha、风险模型、交易成本和组合优化之间的联动。
3. 多因子实务里常用 “IC → 分层 → 正交 → 组合 → 成本容量 → 样本外” 逐层验证。
4. 近期 arXiv q-fin.PM 仍有研究强调可复现实验协议、固定数据切分、交易成本、组合测试和可审计的失败假设，这与 Vortex 的因子档案机制一致。例如 2026-04 的加密货币因子发现研究把 agent 发现约束在可审计 DSL、固定切分、交易成本和组合测试中；2026-04 的 A 股 5 分钟全市场组合优化研究也说明分钟级大 universe 研究必须同时处理计算规模、组合约束和高阶风险。

## 入档要求

每个重要因子档案必须明确它当前属于哪一类角色。如果只是排序/过滤/择时组件，档案中不能写成“策略候选”；如果只是裸跑失败，也不能直接写成“因子无用”。
