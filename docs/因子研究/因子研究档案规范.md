# 因子研究档案规范

关联：[[因子研究档案]]、[[因子评价框架]]、[[因子实验总表]]、[[因子研究与评测全流程说明]]、[[研究协作与产物治理]]

## 目标

因子研究档案用于保存“以后能复用的经验”，不是保存每个临时文件。它要回答：

1. 这个因子假设是什么？
2. 用了哪些数据字段？
3. 是否有预测力？
4. 为什么能用或不能用？
5. 后续是否值得继续？

## 入库标准

必须建档：

1. 通过准入或可能进入策略组合。
2. 明确失败但提供可复用经验。
3. 发现未来函数、PIT、复权、成交口径、数据单位等关键问题。
4. 出现“IC 有效但策略无效”的重要现象。
5. 用户或研究员明确要求保留。

入库时必须同步标记因子角色，参考 [[因子评价框架]]：

1. Alpha 排序因子。
2. 过滤因子。
3. 择时/状态因子。
4. 仓位因子。
5. 风险因子。
6. 组合腿。

不建档：

1. 命令失败但无研究结论。
2. 只是路径、格式、依赖排障。
3. 重复参数微调，没有新增经验。
4. 没有数据、没有指标、没有下一步。

## 状态机

| 状态 | 含义 |
|---|---|
| `raw_idea` | 只有假设，未系统验证 |
| `evaluated` | 已完成基础 IC/多空评测 |
| `research_lead` | 有线索但还不能进策略 |
| `candidate` | 值得拥有，进入候选加固 |
| `rejected` | 明确不采用，保留失败经验 |
| `invalid_experiment` | 数据口径不可信，结果不可引用 |
| `promoted` | 已产品化为正式信号或策略组件 |

## 仓库边界

进入仓库：

1. 结论、证据、边界、下一步。
2. 少量关键指标表。
3. workspace artifact 路径。
4. Obsidian 链接。

不进入仓库：

1. 大 CSV。
2. 大 HTML。
3. 大 JSON。
4. 参数网格全集。
5. 每日持仓明细。
