# 05-08 监督台启发式评审

评审日期：2026-05-17
评审范围：交易台、数据服务、设置 / 集成管理、运行输出
评审框架：Nielsen 十项可用性原则，结合单人量化研发公司的日常监督场景。

## 结论

当前真正高频入口是 05-08。01-04 更适合保留为 Agent / 研发后台入口，不应默认抢占主导航和首屏。监督台第一原则是回答三件事：现在是否安全、今天是否需要动作、配置和数据是否健康。

## 立即修正

1. 主导航优先级错误
   - 问题：默认进入 01 运营驾驶舱，但用户当前日常任务集中在交易、数据、配置和输出。
   - 影响：每次打开页面都要再次定位到真正关心的监督入口。
   - 修正：左侧导航改为“日常监督”优先，默认进入交易台；01-04 降级为“研发 / Agent 入口”。

2. 设置页信息密度过高
   - 问题：脱敏变量表格默认全部展开，挤压真正可编辑的配置项。
   - 影响：配置任务被审计信息淹没。
   - 修正：脱敏变量改为折叠诊断信息，保留当前状态、输入框和测试按钮。

3. 数据服务缺少进度感
   - 问题：刷新后即使状态已是最新，也难以确认是否真的完成；数据抓取进度没有视觉锚点。
   - 影响：用户需要读多段文本判断数据是否可用。
   - 修正：新增进度条，突出“更新进度”卡片，保留最近完成数据集列表。

## 后续建议

1. 交易台可以进一步拆成“今日动作”和“账户复盘”两个折叠层级，避免当前动作被历史审计信息打断。
2. 数据服务页后续应接入运行日志 tail 和 dataset 级失败原因，而不是只展示完成结果。
3. 运行输出页应长期定位为调试面板；常规用户旅程不应依赖它阅读 JSON。
4. 01-04 可以保留，但未来更适合变成 Agent 工作流入口，而不是和监督台同级竞争注意力。
